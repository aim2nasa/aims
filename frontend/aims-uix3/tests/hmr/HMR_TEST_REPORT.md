## 🔧 4. 해결 방법 (기술적 세부사항)

### 커스텀 Vite 플러그인 작성

**파일**: `vite-plugins/css-reload-plugin.js`

```javascript
export default function cssReloadPlugin() {
  return {
    name: 'css-reload-plugin',
    handleHotUpdate({ file, server }) {
      if (file.endsWith('.css')) {
        console.log(`[CSS-Reload] ${file} changed - triggering full reload`)
        server.ws.send({ type: 'full-reload', path: '*' })
        return []
      }
    }
  }
}
```

### Vite 설정 변경

**파일**: `vite.config.ts`

#### 변경 전 (문제 있던 설정)
```typescript
watch: {
  ignored: [
    '**/node_modules/**',
    '**/.git/**',
    '**/.vscode/**',
    '**/*.css'  // ❌ CSS 파일 감시 안 함!
  ]
}
```

#### 변경 후 (해결된 설정)
```typescript
plugins: [
  react(),
  tsconfigPaths(),
  cssReloadPlugin()  // ✅ CSS 리로드 플러그인 추가
],
watch: {
  ignored: [
    '**/node_modules/**',
    '**/.git/**',
    '**/.vscode/**'
    // ✅ CSS 파일 감시 활성화!
  ]
}
```

### 작동 원리

1. **CSS 파일 변경 감지**
   - Vite가 CSS 파일 변경 감지
   - 커스텀 플러그인의 `handleHotUpdate` 훅 실행

2. **전체 페이지 리로드 강제**
   - WebSocket으로 브라우저에 리로드 명령 전송
   - 2-3초 내 브라우저 자동 새로고침

3. **안정성 보장**
   - Vite 기본 CSS HMR 대신 전체 리로드 사용
   - Windows 파일 시스템에서 100% 안정적
