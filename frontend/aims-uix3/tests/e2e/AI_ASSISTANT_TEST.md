# AI 어시스턴트 E2E 테스트

## 개요

aims-uix3 프론트엔드의 AI 어시스턴트 기능을 검증하는 Playwright E2E 테스트입니다.

**Maestro (모바일)와 달리 Playwright는 한글(Unicode) 입력을 완벽하게 지원**하여 AI 어시스턴트와의 깊은 대화 테스트가 가능합니다.

## 테스트 실행

```bash
# 전체 테스트 실행
cd frontend/aims-uix3
npx playwright test tests/e2e/ai-assistant.spec.ts

# 특정 테스트만 실행
npx playwright test tests/e2e/ai-assistant.spec.ts --grep "고객 조회"

# UI 모드로 실행 (디버깅)
npx playwright test tests/e2e/ai-assistant.spec.ts --ui
```

## 핵심 헬퍼 함수

### `loginAndSetup(page)`
- 개발자 모드 활성화 (Ctrl+Alt+Shift+D)
- 개발용 로그인 건너뛰기 버튼 클릭
- 온보딩 완료 상태 설정 (localStorage)

### `openChatPanel(page)`
- 헤더의 AI 어시스턴트 버튼 클릭
- 채팅 패널 열림 대기
- 온보딩 오버레이 자동 닫기

### `sendMessageAndWaitResponse(page, message)`
- 입력창에 한글 메시지 입력
- Enter 키로 전송
- AI 응답 대기 (최대 60초)
- 응답 텍스트 반환

### `dismissOnboarding(page)`
- 온보딩 투어 오버레이 자동 닫기
- "다시 표시 안 함" 버튼 우선 클릭
- ESC 키 폴백

## 9개 기능 카테고리

AI 어시스턴트가 지원하는 기능 카테고리:

1. **고객 조회** - 검색, 지역별, 유형별, 상세 정보
2. **고객 등록** - 개인/법인 고객 추가
3. **고객별 문서 조회** - 특정 고객의 문서 목록
4. **고객 정보 수정** - 연락처, 주소 등 수정
5. **계약 조회** - 목록, 상세, 피보험자 조회
6. **생일 고객** - 특정 월/일의 생일 고객
7. **문서 검색** - 키워드 + AI 의미 통합 검색
8. **고객 메모** - 메모 추가, 조회, 삭제
9. **고객 관계** - 관계 조회 및 등록

## 트러블슈팅

### 온보딩 오버레이가 클릭을 막는 경우
- `dismissOnboarding(page)` 함수가 자동으로 처리
- localStorage에 `aims_onboarding_completed: true` 설정

### AI 응답 대기 시간 초과
- 기본 타임아웃: 60초
- MCP 서버 상태 확인 필요

### 개발자 로그인 버튼이 보이지 않는 경우
- Ctrl+Alt+Shift+D로 개발자 모드 활성화 필요
