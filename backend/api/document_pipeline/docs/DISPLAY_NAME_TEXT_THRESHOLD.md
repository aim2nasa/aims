# DisplayName 생성 텍스트 최소량 기준

## 배경

2026-03-24, 텍스트가 없는 이미지(비행기 부품 사진, DSCN6461.jpg)를 업로드했을 때
AI가 "곽승철 삼성화재 진단서 2024.03"이라는 엉뚱한 별칭을 생성하는 버그 발생.

### 원인

OCR이 이미지에서 추출한 텍스트가 "◆" 1글자(confidence 49.9%)였으나,
`ocr_worker.py`의 `summarize_text` 호출 시 텍스트 길이 체크 없이 AI에 전달.
AI가 고객명(곽승철) + 문서 유형(diagnosis) 정보를 조합하여 환각 별칭을 생성.

## 기준: 텍스트 절대량 (10자 이상)

### 왜 OCR 신뢰도가 아닌 텍스트 절대량인가

| 기준 | 장점 | 단점 |
|------|------|------|
| OCR 신뢰도 | 간접적 품질 지표 | 신뢰도 0.5 + 텍스트 500자 → 유효한데 차단됨 |
| **텍스트 절대량** | **AI 입력의 직접적 품질 지표** | — |

AI가 의미 있는 별칭을 생성하려면 **텍스트 자체에 충분한 정보가 있어야** 한다.
OCR 신뢰도는 글자 인식 정확도의 간접 지표일 뿐, 텍스트 유용성과 직접 관련이 없다.

실제 DB 데이터 분석 (2026-03-24):

| OCR 신뢰도 구간 | 문서 수 | 평균 텍스트 길이 |
|----------------|--------|---------------|
| 0.0~0.1 | 7 | 0자 |
| 0.2~0.3 | 2 | 1자 |
| 0.4~0.5 | 4 | 68자 |
| 0.5~0.6 | 3 | 937자 |
| 0.6 이상 | 523 | 3,000~15,000자 |

→ 낮은 신뢰도에서도 텍스트가 충분하면 AI가 활용 가능.
→ 높은 신뢰도에서도 텍스트가 부족하면 환각 발생 가능.

### 임계값: 10자

- `len(text.strip()) < 10` 이면 AI 호출 스킵
- 10자 미만의 텍스트로는 문서 제목의 핵심 정보(이름, 날짜, 기관명 등)를 추출할 수 없음
- displayName 미생성 시 프론트엔드는 원본 파일명(`originalName`)을 표시
