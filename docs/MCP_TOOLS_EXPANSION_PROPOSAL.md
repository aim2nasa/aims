# AIMS MCP 도구 확장 제안

> 작성일: 2026-02-26
> 현재 도구 수: 34개 → 확장 후 예상: 44개+

---

## 현황

AIMS MCP 서버(`aims_mcp`)는 1개 서버에 34개 도구를 제공합니다.
현재 커버: 고객 CRUD, 계약 조회, 문서 검색, AR/CRS, 메모, 관계, 생일, RAG, 통합검색, 유틸리티.

---

## 1단계: 높은 가치 (설계사 핵심 업무 직결)

### 1-1. 보장 분석 (`analyze_coverage_gap`)

**사용 예시**: "김철수 고객의 보장이 부족한 부분은?"

- AR에서 파싱된 계약 데이터(보험종류, 가입금액, 보장내용)를 기반으로 보장 빈 곳(gap) 분석
- 사망, 암, 실손, 후유장해, 입원 등 주요 보장 항목별 현황 요약
- 기존 데이터: `annual_reports` + `contracts` 컬렉션
- **DGX Spark 도입 후 AI 분석 품질 향상 가능** (현재는 규칙 기반으로도 구현 가능)

### 1-2. 가족 통합 보장 현황 (`get_family_coverage`)

**사용 예시**: "김철수 가족 전체 보험 현황 정리해줘"

- `relationships` 데이터에서 가족 관계 추출
- 가족 구성원별 계약 목록 + 총 보험료 + 보장 요약을 한 번에 반환
- 설계사가 가장 빈번하게 수행하는 작업 중 하나
- 기존 도구 조합: `list_relationships` → 각 구성원 `get_annual_reports` (현재 여러 번 호출 필요)

### 1-3. 만기 도래 계약 조회 (`find_expiring_contracts`)

**사용 예시**: "3개월 내 만기 도래하는 계약 알려줘"

- AR/CRS에서 파싱된 계약 만기일 기준으로 필터링
- 갱신형 vs 비갱신형 구분, 보험료 변동 예상 정보 포함
- 갱신/재설계 기회 → 매출에 직결
- 파라미터: `months_ahead` (기본 3개월)

### 1-4. 미팅 브리핑 생성 (`generate_meeting_brief`)

**사용 예시**: "내일 만날 김철수 고객 브리핑 만들어줘"

- 고객 기본정보 + 최근 계약 변동 + 메모 + 생일/기념일 + 관계 네트워크를 한 장으로 요약
- 기존 도구 5~6개를 호출해야 하는 작업을 1회 호출로 압축
- LLM이 자연어 요약을 생성하되, MCP 도구는 데이터 수집 + 구조화 담당
- **설계사 체감 가치가 가장 큰 기능** — "AIMS가 나 대신 준비해줬다"

## 2단계: 중간 가치 (업무 효율화)

### 2-1. 포트폴리오 통계 (`get_portfolio_stats`)

**사용 예시**: "내 전체 고객 수, 총 보험료, 보험사별 비중은?"

- 전체 고객 수, 활성/휴면 비율
- 총 보험료 (월납/연납 환산)
- 보험사별 계약 비중
- 신규 고객 추이 (월별)

### 2-2. 계약 변동 감지 (`detect_contract_changes`)

**사용 예시**: "최근 해약/감액된 계약 있어?"

- 동일 고객의 연속 AR을 비교하여 변동 사항 자동 감지
- 감지 항목: 해약, 감액, 실효, 보험료 변경, 신규 가입
- 기존 `get_ar_contract_history`의 확장 버전 — 변동만 필터링
- 고객 이탈 방지의 핵심 도구

### 2-3. 고객 세그먼트 조회 (`search_customer_segments`)

**사용 예시**: "VIP 고객 목록", "6개월 이상 미접촉 고객"

- 조건 기반 고객 필터링
  - 보험료 기준 (월 50만원 이상 등)
  - 최근 접촉일 기준 (메모 최신 날짜)
  - 계약 수 기준
  - 지역 기준
- 타겟 마케팅·관리를 위한 세그먼테이션

### 2-4. 문서 업로드 요청 (`request_document_upload`)

**사용 예시**: "이 PDF를 김철수 고객에 등록해줘"

- 채팅 중 파일 첨부 → 특정 고객에 문서 연결
- document_pipeline API 호출 트리거
- 현재는 웹 UI에서만 업로드 가능 → 채팅에서도 가능하게

## 3단계: DGX Spark 도입 후 확장

### 3-1. 고객 인사이트 리포트 (`generate_customer_insight`)

**사용 예시**: "김철수 고객 종합 분석 리포트 생성해줘"

- 전체 계약 이력 + 보장 분석 + 보험료 추이 + 가족 현황 + 메모 종합
- AI가 자연어로 인사이트 작성 (예: "암 보장이 진단금 2천만원으로 부족할 수 있습니다")
- **토큰 소비가 크므로 DGX Spark 로컬 추론 후 구현이 적합**
- PDF 리포트 내보내기 기능 연계

### 3-2. 보험 상품 추천 (`recommend_products`)

**사용 예시**: "50대 남성, 암 가족력, 추천 상품은?"

- 고객 프로필 (나이, 성별, 직업, 건강 이력) + 현재 보장 현황 기반
- `products` 컬렉션의 상품 DB와 매칭
- 보장 gap을 채울 수 있는 상품 우선 추천
- **규제 주의**: "추천"이 아닌 "참고 정보 제공"으로 표현 (불완전판매 이슈)

## 구현 우선순위

```
즉시 구현 가능 (기존 데이터 활용, 규칙 기반)
├── 1-3. 만기 도래 계약 조회
├── 2-1. 포트폴리오 통계
├── 2-2. 계약 변동 감지
└── 2-3. 고객 세그먼트 조회

LLM 활용 필요 (현재 모델로도 가능)
├── 1-1. 보장 분석
├── 1-2. 가족 통합 보장 현황
└── 1-4. 미팅 브리핑 생성

DGX Spark 이후
├── 3-1. 고객 인사이트 리포트
└── 3-2. 보험 상품 추천

별도 인프라 필요
└── 2-4. 문서 업로드 요청 (document_pipeline 연동)
```

## 기대 효과

| 지표 | 현재 (34개 도구) | 확장 후 (44개+ 도구) |
|------|----------------|-------------------|
| 설계사 주요 업무 커버리지 | 조회·검색 중심 | 분석·인사이트까지 확장 |
| 평균 응답 완성도 | 데이터 나열 | 데이터 + 해석 + 액션 제안 |
| 1회 질문으로 완결되는 비율 | ~60% (복합 질문 시 다중 호출 필요) | ~90% (복합 도구로 1회 완결) |
| DGX Spark 활용도 | 기존 도구의 모델 전환 | 신규 고급 AI 기능 추가 |
