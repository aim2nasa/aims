# Annual Report 기반 고객 등록/업데이트 시나리오

> **버전**: v0.5.0
> **작성일**: 2026-01-13
> **상태**: 설계 확정

---

## 1. 개요

Annual Report(AR) 문서가 업로드되면, 첫 페이지를 파싱하여 **피보험자 이름**을 추출한다.
이 이름을 기반으로 고객을 새로 등록하거나, 기존 고객의 AR 정보를 업데이트한다.

---

## 2. 용어 정의

| 용어 | 설명 | 예시 |
|------|------|------|
| **AR 파싱 이름** | AR 문서 첫 페이지에서 추출한 피보험자 이름. AIMS 고객명의 **후보**일 뿐, 실제 고객명이 아닐 수 있음 | `"박형서"` |
| **AIMS 고객명** | DB에 저장된 실제 고객명. 같은 설계사 내에서 **유일성 보장** | `"박형서"`, `"박형서1"`, `"박형서 일산동구"` |

### 핵심 원칙

```
AR 파싱 이름 ≠ AIMS 고객명
AR 파싱 이름 = 검색 키워드 (후보 이름)
```

### 검색 방식

DB에서 **"박형서"가 포함된 모든 고객**을 검색한다. (부분 일치)

```
검색어: "박형서"

✅ 검색됨: "박형서"
✅ 검색됨: "박형서1"
✅ 검색됨: "박형서 일산동구"
✅ 검색됨: "일산 박형서"
✅ 검색됨: "2024 박형서"

❌ 검색 안됨: "박영수" (다른 이름)
```

> **이유**: 설계사가 고유한 이름을 만들 때 `"일산 박형서"`, `"2024 박형서"` 등 다양한 형식을 사용할 수 있기 때문

## 4. 시나리오 상세

### Case 1: 유사 이름의 고객이 없는 경우

**조건**: DB에 "박형서"가 포함된 고객이 0명

**처리** (자동):
1. "박형서" 이름으로 새 고객 **자동 등록**
2. AR 분석 결과를 해당 고객에 연결하여 저장

> **Note**: PDF 파서로 정확히 추출하므로 이름 오류 없음

**결과**:
- 새 고객 `"박형서"` 생성
- AR 데이터 저장 완료

---

### Case 2: 유사 이름의 고객이 존재하는 경우

**조건**: DB에 "박형서"가 포함된 고객이 1명 이상

**예시 상황**:
```
DB 검색 결과: ["박형서", "박형서1", "박형서 일산동구", "일산 박형서", "2024 박형서"]
```

**UI 표시**: 기존 고객 목록 + 세부 정보 (구분을 위한 추가 정보)

---

#### Case 2.1: 기존 고객 선택

**상황**: 설계사가 목록에서 기존 고객을 선택

**처리**:
1. 선택된 고객 (예: `"박형서 일산동구"`)에 AR 추가
2. 동일 발행일 AR 없음 → 새 AR로 추가
3. 동일 발행일 AR 있음 → 기존 AR 갱신

**결과**:
- 기존 고객의 AR 히스토리에 추가 완료

---

#### Case 2.2: 새 고객 등록

**상황**: 설계사가 "새 고객으로 등록" 선택

**처리**:
1. 새 고객명 입력 요청 (예: `"박형서 부산"`)
2. 고객명 유일성 검증
3. 새 고객 등록 + AR 분석 결과 저장

**결과**:
- 새 고객 `"박형서 부산"` 생성
- AR 데이터 저장 완료

## 5. 고객명 유일성 규칙

> **AIMS 철칙**: 같은 설계사(userId) 내에서 고객명은 **절대 중복 불가**

| 검증 항목 | 규칙 |
|-----------|------|
| 개인/법인 구분 | 중복 불가 |
| 활성/휴면 상태 | 중복 불가 |
| 대소문자 | 동일 취급 (중복 불가) |

**예시**:
```
✅ 허용: 설계사A "박형서" + 설계사B "박형서" (다른 설계사)
❌ 금지: 설계사A "박형서" + 설계사A "박형서" (같은 설계사)
✅ 허용: 설계사A "박형서" + 설계사A "박형서 부산" (다른 이름)
```

## 6. AR 히스토리 관리

### 핵심 원칙

한 고객은 **여러 개의 AR 문서**를 가질 수 있으며, **발행일**로 구분된다.

```
고객: 박형서
├── AR (2011.01.01) - 계약 2건
├── AR (2015.03.01) - 계약 3건
├── AR (2021.12.23) - 계약 5건
└── AR (2026.01.13) - 계약 6건  ← 최신
```

### 저장 방식

| 항목 | 설명 |
|------|------|
| **덮어쓰기 ❌** | AR은 덮어쓰지 않음 |
| **히스토리 ✅** | 발행일별로 모든 AR 스냅샷 보관 |
| **추적 가능** | 시간에 따른 계약 변화 추적 |

### AR 문서 구조 (발행일별)

```
AR 문서 (발행일: 2025.08.29)
├── 고객명: 김보성
├── 발행일: 2025.08.29
├── 총 월보험료: 1,809,150원
├── 계약 건수: 6건
└── 보험 계약 목록
    ├── 증권번호: 0004155605, 상품: 무배당 마스터플랜..., 계약일: 2009-06-10
    ├── 증권번호: 0011533898, 상품: 무배당 실버플랜..., 계약일: 2014-03-21
    └── ... (6건)
```

### 새 AR 업로드 시

| 상황 | 처리 |
|------|------|
| 동일 발행일 AR 없음 | 새 AR로 추가 |
| 동일 발행일 AR 있음 | 기존 AR 갱신 (같은 날짜는 1개만) |

> **Note**: 현재 스코프에서 보험사는 **메트라이프**만 지원

## 7. UI/UX 고려사항

### AR 히스토리 UI (RightPane)

#### Step 1: Annual Report 목록

RightPane 하단의 Annual Report 섹션에서 **고객별 최신 AR** 표시:

```
┌─ Annual Report ─────────────────────────────────────────┐
│ 소유주     발행일      파싱일시           총 월보험료   계약 │
├─────────────────────────────────────────────────────────┤
│ 정부군     2025.08.29  2026.01.12 17:53  294,170원    4건 │
│ 신상철     2025.08.29  2026.01.12 17:53  1,115,626원  4건 │
│ 김보성     2025.08.29  2026.01.12 17:53  1,809,150원  6건 │ ← 클릭
│ 안영미     2025.08.27  2026.01.12 17:53  14,102,137원 10건│
└─────────────────────────────────────────────────────────┘
```

#### Step 2: 고객의 AR 히스토리

고객(김보성) 클릭 시 → **해당 고객의 발행일별 AR 목록** 표시:

```
┌─ 김보성님의 AR 히스토리 ──────────────────────────────────┐
│                                          [← 목록으로]   │
├─────────────────────────────────────────────────────────┤
│ 발행일      총 월보험료    계약 건수                      │
├─────────────────────────────────────────────────────────┤
│ 2025.08.29  1,809,150원   6건  ← 최신                   │
│ 2024.01.15  1,500,000원   5건                           │
│ 2023.06.20  1,200,000원   4건                           │
│ 2022.03.10    950,000원   3건                           │
└─────────────────────────────────────────────────────────┘
```

#### Step 3: AR 상세 보기 (모달)

특정 발행일 클릭 시 → **AR 상세 모달** + 상단에 발행일 드롭다운:

```
┌─ 김보성님의 Annual Report ───────────────────────── □ ✕ ─┐
│ 발행일: [2025.08.29 ▼]  ← 드롭다운으로 다른 발행일 선택   │
│         ├─ 2025.08.29 (현재)                            │
│         ├─ 2024.01.15                                   │
│         ├─ 2023.06.20                                   │
│         └─ 2022.03.10                                   │
├─────────────────────────────────────────────────────────┤
│ 발행일: 2025.08.29                                      │
│ 총 월보험료: 1,809,150원                                 │
│ 계약 건수: 6건                                          │
├─────────────────────────────────────────────────────────┤
│ 보험 계약 목록 (6건)                                     │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ 증권번호    상품명              계약일    보험료     │ │
│ │ 0004155605  무배당 마스터플랜...  2009-06-10  65,200원│ │
```

### 고객 선택 화면 (Case 2)

설계사가 올바른 고객을 선택할 수 있도록 **충분한 구분 정보** 제공:

| 표시 정보 | 예시 |
|-----------|------|
| 고객명 | 박형서 일산동구 |
| 등록일 | 2025.03.15 |
| 최근 AR 날짜 | 2025.12.01 |
| 계약 수 | 3건 |
| 메모/태그 | "메트라이프 주력" |

### 새 고객명 입력 (Case 2.2)

- 기존 고객명과 중복되지 않도록 **실시간 유효성 검사**
- 추천 형식 제안: `"박형서 [지역]"`, `"박형서 [특징]"`
