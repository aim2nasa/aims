# MetLife 고객목록조회 자동화

## 개요

`MetlifeCustomerList.py`는 MetLife Digital Office의 고객목록조회 페이지에서 고객을 순차적으로 처리하는 SikuliX 스크립트입니다.

## 실행 방법

```powershell
cd D:\aims\tools\MetlifePDF.sikuli
.\run_customerlist.ps1
```

## 핵심 설정값

| 설정 | 값 | 설명 |
|------|-----|------|
| `FIRST_ROW_OFFSET` | 40px | 헤더에서 첫 번째 행까지 거리 |
| `ROW_HEIGHT` | 33px | 행 간 간격 |
| `CUSTOMERS_PER_PAGE` | 16 | 한 페이지에 보이는 고객 수 |
| `SCROLL_CLICKS` | 5 | 스크롤 휠 클릭 수 (캘리브레이션 필요) |

### 오프셋 계산 공식

```python
offset_y = FIRST_ROW_OFFSET + (ROW_HEIGHT * row)
# row 0: 40px
# row 1: 73px
# row 2: 106px
# ...
# row 15: 535px
```

## 처리 흐름

```
1단계: 고객목록조회 진입
  ├─ 메인 화면 이동
  ├─ 고객관리 클릭
  ├─ 고객등록 클릭
  └─ 고객목록조회 클릭

2단계: 초성별 고객 처리
  ├─ 초성 버튼 클릭 (ㄱ, ㄴ, ㄷ, ...)
  ├─ 고객명 내림차순 정렬
  └─ while (스크롤 끝 아님):
       ├─ 16명 고객 처리:
       │    ├─ 고객 클릭 (offset 계산)
       │    ├─ 고객등록/조회 페이지 대기
       │    └─ 종료(x) 버튼 클릭
       ├─ 스크롤 끝 확인
       └─ 스크롤 다운
```

## 스크롤 로직

### 철칙
1. **고객 누락 절대 금지** - 한 명도 빠뜨리면 안 됨
2. **중복 허용** - 파일 저장 시 중복 파일은 스킵하면 됨
3. **스크롤 끝 감지** - `IMG_SCROLL_BOTTOM` 이미지로 감지

### 스크롤 캘리브레이션
- `SCROLL_CLICKS = 5`로 설정 시 약 15행 스크롤
- 16행 처리 후 15행 스크롤 → 1행 중복 (안전)
- 휠 클릭당 스크롤 양은 시스템마다 다를 수 있음

## 태그

- `metlife-customerlist-v1.0`: 16명 순차 처리 완성 (스크롤 없음)

## 주의사항

1. **전체화면 모드**에서 테스트 필수 (오프셋 값이 달라질 수 있음)
2. 스크립트 실행 전 MetLife Digital Office가 로그인된 상태여야 함
3. 화면 해상도 변경 시 이미지 재캡처 필요

## 예상 소요 시간

- ㄱ 초성 200명: 약 27분 (중복 포함)
- 전체 고객 (추정): 초성별 고객 수에 따라 다름
