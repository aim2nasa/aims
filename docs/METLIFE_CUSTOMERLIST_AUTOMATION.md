# MetLife 고객목록조회 자동화

## 개요

`MetlifeCustomerList.py`는 MetLife Digital Office의 고객목록조회 페이지에서 고객을 순차적으로 처리하는 SikuliX 스크립트입니다.

## 실행 방법

```powershell
cd D:\aims\tools\MetlifePDF.sikuli
.\run_customerlist.ps1
```

## 핵심 설정값

| 설정 | 값 | 설명 |
|------|-----|------|
| `FIRST_ROW_OFFSET` | 40px | 헤더에서 첫 번째 행까지 거리 |
| `ROW_HEIGHT` | 33px | 행 간 간격 |
| `MAX_CUSTOMERS_PER_PAGE` | 15 | OCR로 인식하는 행 수 (마지막 행 잘림) |
| `SCROLL_CLICKS` | 5 | 스크롤 휠 클릭 수 (캘리브레이션 필요) |
| `CAPTURE_DIR` | `D:\captures\metlife_ocr` | 캡처/OCR 결과 저장 위치 |

### 오프셋 계산 공식

```python
offset_y = FIRST_ROW_OFFSET + (ROW_HEIGHT * row)
# row 0: 40px
# row 1: 73px
# row 2: 106px
# ...
# row 15: 535px
```

## 처리 흐름

```
1단계: 고객목록조회 진입
  ├─ 메인 화면 이동
  ├─ 고객관리 클릭
  ├─ 고객등록 클릭
  └─ 고객목록조회 클릭

2단계: 초성별 고객 처리
  ├─ 초성 버튼 클릭 (ㄱ, ㄴ, ㄷ, ...)
  ├─ 고객명 내림차순 정렬
  └─ while (스크롤 끝 아님):
       ├─ 16명 고객 처리:
       │    ├─ 고객 클릭 (offset 계산)
       │    ├─ 고객등록/조회 페이지 대기
       │    └─ 종료(x) 버튼 클릭
       ├─ 스크롤 끝 확인
       └─ 스크롤 다운
```

## 스크롤 로직

### 철칙
1. **고객 누락 절대 금지** - 한 명도 빠뜨리면 안 됨
2. **중복 허용** - 파일 저장 시 중복 파일은 스킵하면 됨
3. **스크롤 끝 감지** - `IMG_SCROLL_BOTTOM` 이미지로 감지

### 스크롤 캘리브레이션

**테스트 결과 (2026-01-24)**:
- `SCROLL_CLICKS = 5` → 정확히 15행 스크롤
- 스크롤 전: 강정모 (15번) 마지막
- 스크롤 후: 강지선 (16번)이 첫 번째 행에 표시

**스크롤 방법**:
```python
# 스크롤바 영역 클릭 후 휠 스크롤 (데이터 행 클릭 방지)
scrollbar_area = header.right(1300).below(300)
click(scrollbar_area)   # 포커스
wheel(scrollbar_area, WHEEL_DOWN, 5)  # 15행 스크롤
```

**주의사항**:
- 데이터 행 직접 클릭 시 고객 상세 페이지로 이동함
- 스크롤바 영역 클릭으로 포커스를 주고 휠 스크롤해야 함
- 16행 처리 후 15행 스크롤 → 1행 중복 (안전)
- 휠 클릭당 스크롤 양은 시스템마다 다를 수 있음

**캘리브레이션 테스트 스크립트**:
```powershell
.\run_scroll_test.ps1 -ScrollClicks 5  # 테스트 실행
```
