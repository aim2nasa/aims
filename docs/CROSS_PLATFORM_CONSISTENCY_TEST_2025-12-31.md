## 3. MCP 도구 선택 패턴

### 도구 선택 우선순위

```javascript
// 1. 고객 문서 조회 (가장 구체적)
/(고객|님).*(문서|파일|서류)|(문서|파일|서류).*(고객|님)/

// 2. 고객 등록
/고객.*(등록|추가|생성|만들)|(등록|추가|생성|만들).*(고객|가입자)/
/새\s*(고객|가입자)|신규\s*(고객|가입자)/

// 3. 문서 검색 (계약 조회보다 먼저)
/(문서|파일|서류|PDF|요약)$/
/(계약서|청구서|신청서|진단서|증권|양식|자료)/
/문서|파일|서류|PDF/i

// 4. 계약 조회
/계약\s*(목록|현황|조회|검색|리스트)|증권번호|보험\s*계약\s*목록/
/(유지|만기|실효|해지|정상)\s*계약/

// 5. 시맨틱 검색 (의미 기반)
/방법|관련|어떻게|처리|결과/ (특정 엔티티 키워드 없을 때)

// 6. 고객 검색
/고객|가입자|정보/

// 7. 일반 검색
/찾아|검색/
```

## 4. 실패 케이스 분석

### 4.1 계약 조회 실패 (1건)

| 입력 | 기대 | 실제 | 원인 |
|------|------|------|------|
| "증권번호 조회" | list_contracts | search_documents | "증권" 키워드가 문서 검색 패턴에 매칭 |

**분석**: "증권번호"는 계약 조회, "증권 문서"는 문서 검색으로 구분해야 하나, 현재 패턴은 "증권"만으로 문서 검색을 트리거함. 실제 사용 시 대화 문맥으로 구분 가능.

### 4.2 문서 요약 실패 (20건)

| 입력 | 기대 | 실제 | 원인 |
|------|------|------|------|
| "내용 알려줘" | search_documents | null | 문맥 없이 너무 일반적 |
| "요약 보여줘" | search_documents | null | 문서 키워드 없음 |

**분석**: 이 입력들은 단독으로는 의미가 불명확함. 실제 대화에서는 이전 문맥(특정 문서 언급)이 있어야 의미가 있으며, 그 경우 정상 동작함.

### 4.3 A-B 역할 전환 실패 (1건)

일부 엣지 케이스에서 양쪽 플랫폼 모두 도구 선택에 실패하는 경우 발생. 일관성 측면에서는 "둘 다 실패"도 일관된 동작으로 볼 수 있음.
