# ISSUE-2: 중복 파일 업로드 UX 개선 필요

> **발견일**: 2026-03-08
> **심각도**: Minor (UX)
> **연관 작업**: 캐치업코리아 446건 v4 분류 체계 튜닝 샘플 업로드
> **상태**: 미해결

---

## 설계 의도 (정상 동작)

- `unique_owner_customer_file_hash` 인덱스로 동일 해시 파일의 중복 등록을 방지
- 해시가 동일하면 파일 내용이 100% 동일하므로, 중복 저장은 불필요
- DuplicateDialog 모달이 뜨며 사용자에게 "건너뛰기" 또는 "취소" 선택지 제공
- "덮어쓰기"/"둘 다 유지" 옵션은 의도적으로 없음 (동일 내용이므로 무의미)

## 현재 동작 흐름

1. 업로드 중 파일 해시(SHA-256) 비교
2. 중복 발견 → `DuplicateDialog` 모달 표시
3. 모달 내용: 고객명, 파일명, 크기, 기존 파일 업로드 날짜
4. 선택지: "취소" (전체 취소) / "건너뛰기 (N초)" (10초 자동 타이머)
5. 체크박스: `☐ 다음 중복 파일도 같은 방식으로 처리 (묻지 않음)`
6. 10초 타이머 만료 시 자동으로 "건너뛰기 + 전체 적용" 처리

## UX 문제점

### 1. 중복 처리 결과 요약 부재

446건 업로드 시 59건이 중복으로 건너뛰어졌으나, 사용자에게 이 사실이 명확히 전달되지 않음.
- `remainingCount` prop이 정의되어 있으나 다이얼로그 UI에 표시되지 않음
- 업로드 완료 후 "59건 중복 건너뜀"과 같은 요약 정보 없음
- 사용자는 446건 업로드했는데 387건만 등록된 이유를 알기 어려움

### 2. 모달 자동 스킵이 너무 빠름

- 10초 자동 타이머로 인해 대량 업로드 시 첫 모달만 잠깐 보이고 나머지는 자동 처리
- 사용자가 중복 파일이 몇 건인지 인지하지 못한 채 업로드가 완료됨

### 3. 중복 파일 목록 미제공

- 어떤 파일이 중복으로 건너뛰어졌는지 확인할 방법이 없음
- 다른 폴더(다른 분류)에 동일 내용 파일이 있는 경우, 어느 쪽이 등록되었는지 파악 불가

## 개선 방향

1. **업로드 완료 요약에 중복 통계 표시**: "총 446건 중 387건 등록, 59건 중복 건너뜀"
2. **건너뛴 파일 목록 제공**: 중복으로 스킵된 파일명 리스트 표시 (접기/펼치기)
3. **모달에 남은 중복 건수 표시**: `remainingCount` prop 활용하여 "남은 중복 파일: N건" 표시
