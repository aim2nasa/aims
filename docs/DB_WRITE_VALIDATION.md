# DB 쓰기 무결성 보호: 대화 시작점 개선

**작성일**: 2025-12-22

---

## 배경

AI 채팅 패널의 기능 메뉴에서 "예시 문구"가 더미 데이터를 포함하고 있어, 사용자가 그대로 전송 시 DB에 쓰레기 데이터가 저장될 위험이 있었음.

### 문제 상황
```
기존 예시: "새 고객 등록해줘. 이름은 홍길동, 전화번호 010-1234-5678"
→ 사용자가 그대로 전송 시 "홍길동" 고객이 DB에 등록됨
```

---

## 해결 방안

### 핵심 변경
**"예시 문구" → "대화 시작점(Conversation Starter)"으로 개념 변경**

| 기존 | 변경 |
|------|------|
| 예시 문구 (따라하기) | 대화 시작점 |
| 그대로 복사해서 사용 | AI와 대화의 출발점 |
| 더미 데이터 포함 | 의도만 전달, 세부사항은 대화로 |

### 장점
1. **안전**: 더미 데이터가 DB에 저장될 위험 없음
2. **자연스러움**: 대화형 인터페이스에 맞는 UX
3. **유연성**: 사용자가 실제 정보를 단계별로 입력
4. **실수 방지**: AI가 확인 후 진행

---

## 구현 내용

### 1. Frontend 대화 시작점 변경 (ChatPanel.tsx)

#### 고객 등록 (➕)
| 변경 전 | 변경 후 |
|---------|---------|
| 새 고객 등록해줘. 이름은 홍길동, 전화번호 010-1234-5678 | 새 고객 등록해줘 |
| 법인 고객 등록해줘. 회사명 ABC주식회사 | 법인 고객 등록해줘 |
| 김영희 고객 추가해줘. 생년월일 1990-05-15 | 새 고객 추가해줘 |

#### 고객 수정 (✏️)
| 변경 전 | 변경 후 |
|---------|---------|
| 홍길동 고객 전화번호를 010-9999-8888로 변경해줘 | 고객 전화번호 수정해줘 |
| 김철수 고객 이메일 주소 수정해줘 | 고객 이메일 수정해줘 |

#### 메모 추가 (📝)
| 변경 전 | 변경 후 |
|---------|---------|
| 이 고객에게 메모 추가해줘: 다음 주 상담 예정 | 이 고객에게 메모 추가해줘 |
| 홍길동 고객 메모: 보험료 인상 안내 완료 | 고객 메모 추가해줘 |

### 2. AI 시스템 프롬프트 업데이트

DB 쓰기 작업 시 대화식 확인 규칙 추가:
- 고객 등록: 이름, 전화번호 등 순차적 확인
- 고객 수정: 대상 고객 확인 → 수정 내용 확인
- 메모 추가: 고객 확인 → 메모 내용 확인

## 예상 대화 흐름

### 고객 등록
```
사용자: 새 고객 등록해줘
AI: 새 고객을 등록하겠습니다. 고객 이름을 알려주세요.
사용자: 김민수
AI: 전화번호도 알려주세요.
사용자: 010-1111-2222
AI: ✅ 김민수 고객(010-1111-2222)을 등록했습니다.
```

### 고객 수정
```
사용자: 고객 전화번호 수정해줘
AI: 어느 고객의 전화번호를 수정할까요?
사용자: 김민수
AI: 김민수 고객을 찾았습니다. 새 전화번호를 알려주세요.
사용자: 010-2222-3333
AI: ✅ 김민수 고객의 전화번호를 010-2222-3333으로 변경했습니다.
```
