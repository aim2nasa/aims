# 요약/전체보기 버튼 비활성화 — 단순화된 문제 정의

> 작성일: 2026-03-27 02:00 KST

---

## 풀고 싶은 문제

요약보기/전체보기 버튼이 **텍스트가 채워질 때만 활성화**되고, 아이콘만 보고도 "이 문서에 텍스트가 있다/없다"를 인식할 수 있게 한다.

| 상태 | 버튼 | 시각적 표시 |
|------|------|-----------|
| 텍스트 없음 | disabled (클릭 불가) | 흐리게 (opacity 0.3) |
| 텍스트 있음 | 활성 (클릭 가능) | 정상 표시 |

---

## 현재 상태

- **코드**: `disabled={!doc._hasMetaText && !doc._hasOcrText}` — 로직 맞음
- **데이터**: React fiber에서 AI/ZIP/JPG 파일의 `_hasMetaText: false`, `_hasOcrText: false` 확인 — 데이터 맞음
- **DOM**: `button.disabled = false` — **반영 안 됨**

---

## 검증해야 할 가설 (우선순위 순)

### 가설 1: 배포된 빌드가 최신이 아님
- 로컬에서 코드는 수정했지만, 실제 서버에 배포된 JS 번들이 이전 버전일 수 있음
- **검증**: 브라우저에서 Ctrl+Shift+R (강력 새로고침) 후 재확인

### 가설 2: 브라우저 캐시
- 이전 빌드의 JS가 캐싱되어 있음
- **검증**: 시크릿 모드에서 테스트

### 가설 3: Playwright evaluate에서 확인한 버튼과 실제 버튼이 다름
- evaluate에서 본 `disabledProp: false`가 tooltip-trigger div의 것일 수 있음
- **검증**: `btn.tagName === 'BUTTON'` + `btn.hasAttribute('disabled')` 확인

## 다음 단계
