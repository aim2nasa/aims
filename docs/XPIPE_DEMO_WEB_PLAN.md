# xPipeWeb — 기획서

**작성일**: 2026-03-19
**목적**: xPipe 엔진의 테스트·검증·성능 측정을 위한 **개발자 전용** 웹 도구
**대상**: 개발자/아키텍트 (일반 대중 배포 안 함, 외부 오픈 안 함)
**위치**: `backend/api/document_pipeline/xpipe/console/web/`
**상태**: 기획 — Agent 리뷰 반영 완료, 구현 대기

---

## 1. 왜 필요한가

xPipe 전략 문서의 **6가지 독립성 원칙 중 3번**:

> **독립 제어**: xPipe 전용 관리 인터페이스(웹/GUI/CUI)를 통해 동작·상태·장애를 모니터링하고, 세부 기능을 제어할 수 있다

현재 상태:
- CLI(`xpipe status/test`) — 개발자용, 실제 체험과 거리가 멂
- 테스트 251개 — 코드 레벨, 눈으로 볼 수 없음
- **실제 문서를 넣고 파이프라인이 동작하는 것을 체험할 방법이 없음**

이 웹 데모가 있어야:
- PR 전에 xPipe가 실제로 동작하는지 검증 가능
- 향후 외부 고객에게 xPipe를 시연할 수 있음
- 어댑터 전환(Insurance ↔ Legal)의 차이를 시각적으로 확인

## 2. AIMS 한계 분석 → xPipe Demo 개선안

### 2.1 AIMS의 5가지 한계

| # | AIMS 한계 | 사용자 영향 | 근본 원인 |
|---|----------|-----------|----------|
| **H1** | **파이프라인이 블랙박스** — 문서를 업로드하면 "처리 중"만 보이고, 내부에서 어떤 단계를 거치는지 전혀 안 보임 | 문제 발생 시 어디서 멈췄는지 모름. 디버깅 불가. 사용자 불안 | 파이프라인 스테이지 정보가 UI에 노출되지 않는 설계 |
| **H2** | **결과가 즉시 안 보임** — 업로드 후 문서 목록에서 개별 클릭해야 분류/감지 결과 확인 가능 | 대량 업로드 시 하나하나 클릭하며 확인해야 함 | 업로드 UI와 결과 UI가 분리된 구조 |
| **H3** | **에러 원인 불투명** — 실패 시 "처리 실패" 한 줄. 어떤 스테이지에서 왜 실패했는지 안 보임 | 재시도 판단 불가 (재시도해도 같은 이유로 또 실패) | 에러 상세가 DB에만 저장, UI에 미노출 |
| **H4** | **비용 감각 없음** — AI API(분류/요약/OCR/임베딩) 호출 비용이 보이지 않음 | 대량 업로드 시 예상치 못한 비용 발생 | CostTracker 없음 |
| **H5** | **처리 방식 고정** — 파이프라인 순서/구성을 변경할 수 없음. OCR 불필요한 문서도 항상 같은 경로 | 불필요한 처리에 시간/비용 소모 | 하드코딩된 파이프라인 |

### 2.2 xPipe Demo의 해결책

| AIMS 한계 | xPipe Demo 해결 | 구현 방식 |
|----------|----------------|----------|
| **H1 블랙박스** | **스테이지별 실시간 진행 표시** — 각 스테이지의 시작/완료/스킵/에러를 실시간으로 보여줌 | EventBus → SSE → UI 실시간 업데이트. 스테이지별 상태 아이콘 + 소요 시간 |
| **H2 결과 지연** | **처리 즉시 결과 인라인 표시** — 파일 목록에서 분류, 감지, 표시명, 품질 점수를 바로 볼 수 있음 | 처리 현황 테이블에 결과 컬럼 포함. 클릭 없이 한눈에 확인 |
| **H3 에러 불투명** | **스테이지별 에러 상세 + 감사 로그** — 어떤 스테이지에서 어떤 이유로 실패했는지 명확히 표시 | AuditLog 뷰어로 문서별 처리 타임라인, 에러 메시지, AI 응답 원문까지 열람 |
| **H4 비용 감각** | **실시간 비용 추적 대시보드** — Provider별 호출 수, 토큰 수, 예상 비용을 실시간 표시 | CostTracker → 하단 바 실시간 업데이트 + 비용 탭 상세 |
| **H5 방식 고정** | **파이프라인 프리셋 전환 + 커스텀 YAML** — UI에서 파이프라인 구성을 변경하여 즉시 적용 | 설정 바에서 프리셋 선택 또는 JSON/YAML 직접 편집 → 다음 업로드부터 적용 |

### 2.3 추가 개선 (AIMS에 없는 기능)

| 기능 | 설명 | 가치 |
|------|------|------|
| **어댑터 전환** | Insurance ↔ Legal 실시간 전환, 같은 문서 재처리 비교 | 이식성 체험 |
| **품질 게이트** | 분류 confidence, 텍스트 품질 자동 평가 + PASS/FAIL | 품질 관리 자동화 |
| **A/B 비교** | 같은 문서를 다른 어댑터/프리셋으로 처리하여 결과 나란히 비교 | 최적 설정 탐색 |
| **제로 인프라** | MongoDB/Redis/Qdrant 없이 `python -m xpipe demo` 한 줄로 시작 | 설치 장벽 제로 |

## 4. 핵심 기능

### 4.1 파일 업로드

| 기능 | 설명 |
|------|------|
| **단일 파일** | 드래그 또는 클릭 |
| **다중 파일** | 여러 파일 동시 드래그/선택 |
| **폴더** | 폴더 드래그 시 내부 파일 전부 처리 |
| **동시 처리** | 큐잉 후 병렬 처리 (설정 가능한 동시성) |
| **지원 형식** | PDF, 이미지, HWP, DOC, TXT 등 |

### 4.2 어댑터 전환

UI에서 어댑터를 실시간 전환:
- **Insurance**: 보험 분류(22개), AR/CRS 감지, 보험 표시명
- **Legal**: 법률 분류(9개), 판결문/계약서 감지, 법률 표시명
- **None (코어만)**: 어댑터 없이 텍스트 추출 + 기본 분류만

같은 문서를 다른 어댑터로 재처리하여 **결과 비교** 가능.

### 4.3 파이프라인 프리셋 전환

- `aims-insurance`: AIMS 기본 파이프라인 (6 스테이지)
- `minimal`: 최소 파이프라인 (Ingest→Extract→Complete)
- **커스텀 JSON/YAML**: 사용자가 직접 파이프라인 정의 입력

### 4.4 실시간 진행률

각 파일의 파이프라인 진행을 **실시간**으로 표시:
- 스테이지별 상태: 대기 → 실행 중 → 완료/스킵/에러
- 소요 시간
- 전체 진행률 바

### 4.5 품질 대시보드

- 파일별 QualityScore (confidence, text_quality, overall)
- PASS/FAIL 뱃지
- 플래그 표시 (LOW_CONFIDENCE, SHORT_TEXT 등)
- 배치 전체 QualityReport (평균, 분포)

### 4.6 성능 측정 (벤치마크)

xPipe의 능력을 시험하고 성능을 정량적으로 측정하는 도구로도 사용한다.

**측정 항목:**

| 지표 | 설명 | 표시 방식 |
|------|------|----------|
| **스테이지별 소요 시간** | 각 스테이지(Ingest/Extract/Classify 등)의 실행 시간 | 파이프라인 상세에 ms 단위 표시 |
| **전체 처리 시간** | 파일 업로드 → Complete까지 총 소요 시간 | 처리 현황 테이블 + 요약 |
| **처리량 (throughput)** | 단위 시간당 처리 문서 수 | 배치 완료 후 "N건/분" 표시 |
| **분류 정확도** | Ground Truth 대비 정확도 (GT 파일 업로드 시) | 배치 결과에 정확도 % 표시 |
| **분류 confidence 분포** | 파일별 confidence → 히스토그램 | 품질 탭에 분포 차트 |
| **비용 효율** | 문서 1건당 평균 비용 | 비용 탭에 "건당 $X.XXX" |
| **에러율** | 실패 문서 비율 | 하단 바 + 배치 요약 |

**벤치마크 모드:**

```
┌─ 벤치마크 결과 ──────────────────────────────────────┐
│                                                       │
│  배치: 50건 처리 완료                                  │
│                                                       │
│  총 소요:     2분 34초                                 │
│  처리량:      19.5건/분                                │
│  평균 시간:   3.1초/건                                 │
│  분류 정확도: 91.8% (GT 제공 시)                       │
│  평균 confidence: 0.87                                │
│  품질 PASS:   48/50 (96%)                             │
│  에러:        0건                                      │
│  총 비용:     $0.15 (건당 $0.003)                     │
│                                                       │
│  스테이지별 평균:                                       │
│  Ingest: 0.02s | Extract: 0.8s | Classify: 1.2s      │
│  Detect: 0.05s | Embed: 2.1s  | Complete: 0.01s      │
│                                                       │
│  병목: Embed (전체의 50.3%)                            │
│                                                       │
│  [CSV 다운로드]  [JSON 다운로드]  [재실행]              │
└───────────────────────────────────────────────────────┘
```

**기능:**
- 배치 완료 시 자동으로 벤치마크 요약 생성
- GT JSON 파일을 함께 업로드하면 분류 정확도 자동 측정
- 스테이지별 병목 식별 (가장 느린 스테이지 하이라이트)
- 결과 CSV/JSON 다운로드 (외부 분석용)
- 같은 배치를 다른 어댑터/프리셋으로 재실행하여 성능 비교

## 5. 기술 구현

### 5.1 아키텍처

```
브라우저 (HTML/JS)
    │
    ▼
xPipe Demo API (FastAPI, 별도 포트 :8200)
    │
    ├── POST /api/upload          — 파일 업로드 + 파이프라인 실행
    ├── POST /api/upload/batch    — 다중 파일 업로드
    ├── GET  /api/status/:id      — 처리 상태 조회
    ├── GET  /api/events          — SSE 실시간 이벤트 스트림
    ├── GET  /api/results/:id     — 처리 결과 조회
    ├── GET  /api/audit/:id       — 감사 로그 조회
    ├── GET  /api/cost            — 비용 요약
    ├── PUT  /api/config          — 어댑터/프리셋 전환
    └── GET  /                    — 웹 UI (단일 HTML)
    │
    ▼
xPipe 엔진 (Pipeline + Adapter + EventBus + AuditLog + QualityGate)
```

### 5.2 프론트엔드

**단일 HTML 파일** (xpipe/console/web/index.html):
- 외부 CDN 의존 없음 (오프라인 동작)
- Vanilla JS + CSS (프레임워크 없음)
- SSE로 실시간 업데이트
- 반응형 레이아웃

이유: xPipe는 독립 패키지이므로, React/Vite 같은 빌드 도구 의존 없이 단일 파일로 제공해야 이식성 원칙에 부합.

### 5.3 백엔드

**`xpipe/console/web/server.py`** (FastAPI):
- xPipe Pipeline 직접 호출
- 인메모리 상태 관리 (처리 중 문서 목록)
- EventBus → SSE 브릿지
- 실제 텍스트 추출은 stub 또는 pdfplumber 직접 사용

### 5.4 실행 방법

```bash
# xPipe Demo 서버 시작
python -m xpipe demo

# 또는 직접
cd backend/api/document_pipeline
python -m xpipe.console.web.server

# 브라우저에서 http://localhost:8200 접속
```

| 항목 | AIMS | xPipe Demo |
|------|------|------------|
| **목적** | 프로덕션 서비스 | 엔진 검증·시연 |

| **품질 점수** | 없음 | **QualityGate 실시간 표시** |
| **비용 추적** | 없음 | **CostTracker 실시간 표시** |
| **감사 로그** | 없음 | **AuditLog 뷰어** |
| **인프라 의존** | MongoDB+Redis+Qdrant | **인메모리 (DB 불필요)** |
| **배포** | 서버+빌드 필요 | **`python -m xpipe demo` 한 줄** |

## 7. 추가 기능

### 7.1 필수 구현

#### 문서 비교 모드 (Side-by-Side)

같은 문서를 **다른 설정(어댑터/프리셋)으로 동시에 처리**하여 결과를 나란히 비교:

```
┌─ Insurance 어댑터 ──────┐  ┌─ Legal 어댑터 ───────────┐
│ 분류: 보험증권           │  │ 분류: 계약서              │
│ 감지: AR (홍길동)        │  │ 감지: 없음                │
│ 표시명: 홍길동_AR_...    │  │ 표시명: 계약서_2026...    │
│ 소요: 3.2초              │  │ 소요: 2.8초               │
│ 비용: $0.003             │  │ 비용: $0.002              │
└──────────────────────────┘  └───────────────────────────┘
```

가치: 어댑터 간 차이를 직관적으로 확인. 새 어댑터 개발 시 기존 대비 품질 비교.

#### 텍스트 하이라이트 뷰어

추출된 텍스트에서 **분류/감지에 기여한 키워드**를 색상별 하이라이트:

```
MetLife                          ← 감지 키워드 (주황)
[홍길동] 고객님을 위한            ← 고객명 추출 (파랑)
[Annual Review Report]           ← 필수 키워드 (빨강)
[보유계약 현황]                   ← 선택 키워드 (초록)
발행(기준)일: [2026년 1월 15일]   ← 날짜 추출 (보라)
```

가치: 분류/감지 근거를 시각적으로 확인. 프롬프트 튜닝 시 필수 도구.

#### 대시보드 요약 (홈)

데모 시작 시 xPipe 엔진의 **전체 상태를 한눈에**:

```
┌─ xPipe Engine ──────────────────────────────┐
│  v0.1.0  |  어댑터: 2개  |  스테이지: 6개    │
│  ┌────────┐  ┌────────┐  ┌────────┐         │
│  │ 오늘    │  │ 품질    │  │ 비용    │         │
│  │ 127건   │  │ 96% ✅  │  │ $0.38  │         │
│  └────────┘  └────────┘  └────────┘         │
│  최근 처리: ...                               │
└──────────────────────────────────────────────┘
```

### 7.2 권장 구현

#### 실패 주입 (Chaos Testing)

특정 스테이지에서 **의도적으로 에러를 발생**시켜 에러 핸들링 검증:

```
설정: [Classify 스테이지에서 50% 확률로 실패]
→ skip_on_error 동작 확인, DLQ 동작 확인, 감사 로그 기록 확인
```

가치: 장애 시나리오를 안전하게 테스트.

#### API 플레이그라운드

xPipe의 개별 기능을 **단독으로 테스트**:

```
┌─ API 테스트 ─────────────────────────────────┐
│  [detect_special_documents]                   │
│  텍스트: [직접 입력 또는 붙여넣기]              │
│  MIME:   [application/pdf ▼]                  │
│  [실행]                                        │
│  결과: [{doc_type: "annual_report", ...}]      │
└───────────────────────────────────────────────┘
```

가치: 전체 파이프라인 없이 개별 어댑터 메서드를 빠르게 테스트.

### 7.3 선택 (추후 필요 시 구현)

### 9.2 보안 관련 (해당 없음 처리)

xPipeWeb은 **개발자 전용, 로컬 전용, 외부 비공개** 도구이므로:
- 인증/인가: 불필요
- CORS: 불필요 (같은 포트에서 HTML 서빙)
- 외부 공격 방어: 해당 없음
- API 키 보호: stub 기본, real 모드는 개발자가 직접 .env 설정
