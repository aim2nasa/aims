# AR 다중 문서 등록 UX 분석 및 개선 계획

> **작성일**: 2026-01-20
> **목적**: 여러 AR 문서 등록 시 UX 문제점 분석 및 개선 방향 정립

---

## 1. 현재 구현 분석

### 1.1 전체 흐름

```
[여러 파일 드롭] → [for 루프 순회] → [AR 감지] → [고객 매칭 분기]
                                                    ↓
                              ┌─────────────────────┴─────────────────────┐
                              ↓                                           ↓
                   [유사 고객 0명]                            [유사 고객 1명+]
                   자동 고객 생성 → 즉시 등록                  큐에 추가 (대기)
                                                                  ↓
                                                    [전체 파일 처리 완료 후]
                                                    큐에서 하나씩 꺼내서
                                                    고객 선택 모달 표시
                                                          ↓
                                                    [선택 완료 → 등록]
                                                          ↓
                                                    processNextArFile()
                                                    (다음 파일 모달)
```

### 1.2 핵심 데이터 구조

```typescript
// 대기 중인 AR 파일 큐
const pendingArFilesQueueRef = useRef<PendingArFile[]>([])

interface PendingArFile {
  file: File                    // 원본 파일
  arMetadata: {
    customer_name: string       // AR에서 추출한 고객명
    issue_date: string          // 발행일
  }
  matchingCustomers: Customer[] // 유사 고객 목록
  fileId: string
  existingHashes: ExistingFileHash[]
}
```

### 1.3 처리 로직 요약

| 단계 | 동작 |
|------|------|
| 1 | 여러 파일 드래그앤드롭 |
| 2 | `for (const file of files)` 루프로 각 파일 처리 |
| 3 | PDF마다 `checkAnnualReportFromPDF()` 호출 |
| 4 | **AR 감지 시 분기**: |
| | - 사전 선택된 고객 있음 → **즉시 등록** |
| | - 유사 고객 0명 → **자동 고객 생성 후 즉시 등록** |
| | - 유사 고객 1명+ → **`pendingArFilesQueueRef` 큐에 추가** |
| 5 | 전체 루프 완료 후 큐에 파일이 있으면 **첫 번째 파일 모달 표시** |
| 6 | 사용자가 고객 선택 → 해당 파일 등록 |
| 7 | `processNextArFile()` 호출 → **다음 파일 모달 표시** (큐 소진까지 반복) |

### 1.4 관련 주요 파일

| 역할 | 경로 |
|------|------|
| 메인 컴포넌트 | `frontend/aims-uix3/src/components/DocumentViews/DocumentRegistrationView/DocumentRegistrationView.tsx` |
| AR 처리 유틸 | `frontend/aims-uix3/src/components/DocumentViews/DocumentRegistrationView/utils/annualReportProcessor.ts` |
| 고객 선택 모달 | `frontend/aims-uix3/src/components/DocumentViews/DocumentRegistrationView/components/CustomerSelectionModal.tsx` |

## 2. UX 문제점 분석

### 2.1 🔴 핵심 문제: 반복적인 모달 피로 (Modal Fatigue)

```
[파일 10개 드롭] → [모달 1] → 선택 → [모달 2] → 선택 → ... → [모달 10] → 선택
```

| 문제 | 설명 |
|------|------|
| **클릭 반복** | 10개 파일 = 최소 10번 클릭 |
| **인지 부하** | 매번 고객 목록을 읽고 판단해야 함 |
| **집중력 저하** | 반복 작업으로 실수 가능성 증가 |

**핵심 인사이트**: AR 문서가 누구에게 매핑되어야 하는지 **한 번 정해지면, 더 이상 사용자 개입이 필요 없다**.

---

### 2.2 🔴 동일 고객 반복 선택

**시나리오**: "홍길동" 고객의 AR 5개를 한 번에 등록

```
현재:   [AR1] "홍길동" 선택 → [AR2] "홍길동" 선택 → [AR3] "홍길동" 선택 → ...
이상적: [AR 5개 모두] → "홍길동" 한 번 선택 → 완료
```

- 같은 고객명이 추출되어도 **매번 동일한 선택을 반복**해야 함
- "이 고객에게 모두 등록" 옵션 없음

---

### 2.3 🔴 진행 상황 불투명

```typescript
// 현재: 콘솔에만 로그 (사용자에게 미표시)
console.log('AR 큐 다음 파일 처리:', nextArFile.file.name, '(남은 파일:', pendingArFilesQueueRef.current.length, '개)');
```

| 문제 | 설명 |
|------|------|
| **남은 개수 미표시** | 모달에 "3/10" 같은 진행률 없음 |
| **소요 시간 예측 불가** | 사용자가 얼마나 더 걸릴지 모름 |
| **전체 파일 목록 미제공** | 어떤 파일들이 대기 중인지 안 보임 |

---

### 2.4 🔴 일괄 처리 옵션 부재

현재 불가능한 작업들:
- ❌ "모두 건너뛰기"
- ❌ "남은 파일 모두 이 고객에게 등록"
- ❌ "모두 새 고객으로 등록"
- ❌ 대기 중인 파일 미리보기/순서 변경

---

### 2.5 🔴 취소 시 피드백 부족

```typescript
onClose={() => {
  setArCustomerSelectionState(prev => ({ ...prev, isOpen: false }));
  // 현재 파일 취소 처리
  updateFileStatusByFile(arFile, 'skipped', '고객 선택 취소');
  processNextArFile(); // 다음 파일로 넘어감
}}
```

| 문제 | 설명 |
|------|------|
| **확인 없이 건너뜀** | X 버튼 = 즉시 skip (경고 없음) |
| **되돌리기 불가** | 건너뛴 파일 복구 옵션 없음 |
| **전체 취소 옵션 없음** | "남은 파일 모두 취소"가 없음 |

---

### 2.6 🔴 컨텍스트 단절

모달이 뜰 때마다:
- 이전에 어떤 고객을 선택했는지 안 보임
- 파일명만 표시되고 **AR 메타데이터(발행일 등) 강조 부족**
- 여러 AR의 관계성 파악 어려움 (같은 고객의 연속 AR인지)

---

### 2.7 🔴 오류 복구 어려움

파싱/등록 실패 시:
- 실패한 파일만 skip하고 계속 진행
- **일괄 재시도 옵션 없음**
- 어떤 파일이 실패했는지 한눈에 파악 어려움

---

## 3. 문제의 본질

> **AR 문서가 누구에게 매핑되어야 하는지가 한 번 정해지면, 더 이상 사용자 개입을 요청할 필요가 없다.**

현재 시스템은 이 원칙을 무시하고:
1. 매 파일마다 개별 확인 요청
2. 같은 고객명이라도 반복 선택 강요
3. 사용자의 의도를 기억/활용하지 않음

---

## 4. 개선 목표

### 4.1 핵심 원칙

1. **한 번 매핑되면 반복 질문 없음**: 같은 고객명 → 같은 고객에게 자동 매핑
2. **일괄 처리 UI**: 여러 고객에게 매핑될 때 한 화면에서 직관적으로 처리
3. **사용자 제어권**: 자동 매핑 결과를 검토하고 수정할 수 있는 기회 제공

### 4.2 이상적인 흐름

```
[여러 AR 드롭]
      ↓
[자동 분석 & 그룹핑]
  - 고객명별로 AR 파일 그룹화
  - 기존 고객과 자동 매칭 시도
      ↓
[일괄 매핑 UI 표시]
  ┌──────────────────────────────────────────────┐
  │  AR 파일 매핑 확인                            │
  │                                              │
  │  📁 홍길동 (3개 파일)     → [홍길동 ▼]        │
  │     - AR_2024_01.pdf                         │
  │     - AR_2024_07.pdf                         │
  │     - AR_2025_01.pdf                         │
  │                                              │
  │  📁 김철수 (2개 파일)     → [새 고객 등록 ▼]  │
  │     - AR_김철수_2024.pdf                     │
  │     - AR_김철수_2025.pdf                     │
  │                                              │
  │  📁 이영희 (1개 파일)     → [이영희A ▼]       │
  │     - AR_이영희_2025.pdf    [이영희B]         │
  │                              [새 고객 등록]   │
  │                                              │
  │            [모두 등록]  [취소]                │
  └──────────────────────────────────────────────┘
      ↓
[확인 한 번으로 일괄 처리]
```

### 4.3 기대 효과

| 현재 | 개선 후 |
|------|--------|
| 10개 파일 = 10번 모달 | 10개 파일 = **1번 확인** |
| 같은 고객 5번 선택 | 같은 고객 **자동 그룹화** |
| 진행률 미표시 | **전체 현황 한눈에 파악** |
| 실수 복구 어려움 | **등록 전 검토 & 수정 가능** |

## 5. 상세 설계

### 5.1 시나리오별 케이스 정의

AR 파일과 고객 매핑은 다음과 같은 조합이 발생할 수 있다:

#### Case A: 단일 파일 → 단일 고객 (1:1)

```
[AR 1개] → [고객 1명]
```

| 상황 | 처리 |
|------|------|
| 정확히 1명 매칭 | 자동 연결 (확인만) |
| 여러 명 매칭 | 선택 필요 |
| 0명 매칭 | 새 고객 생성 |

**현재도 처리 가능** (단, 모달 방식)

---

#### Case B: 다중 파일 → 단일 고객 (N:1)

```
[AR 5개] → [홍길동 1명]
  - AR_홍길동_2023.pdf
  - AR_홍길동_2024_01.pdf
  - AR_홍길동_2024_07.pdf
  - AR_홍길동_2025_01.pdf
  - AR_홍길동_2025_07.pdf
```

**핵심**: 같은 고객명이 추출된 파일들은 **자동 그룹화**하여 한 번의 선택으로 처리

| 상황 | 처리 |
|------|------|
| 모든 파일 → 같은 고객명 | 그룹화 → 1번 선택 → 전체 적용 |
| 정확히 1명 매칭 | 자동 연결 (확인만) |
| 여러 명 매칭 | 그룹 전체에 대해 1번 선택 |

---

#### Case C: 다중 파일 → 다중 고객 (N:N) ⭐ 핵심 케이스

```
[AR 10개] → [고객 4명]

  그룹 1: 홍길동 (3개)
    - AR_홍길동_2024.pdf
    - AR_홍길동_2025_01.pdf
    - AR_홍길동_2025_07.pdf

  그룹 2: 김철수 (2개)
    - AR_김철수_2024.pdf
    - AR_김철수_2025.pdf

  그룹 3: 이영희 (3개)
    - AR_이영희_2023.pdf
    - AR_이영희_2024.pdf
    - AR_이영희_2025.pdf

  그룹 4: 박민수 (2개)
    - AR_박민수_2024.pdf
    - AR_박민수_2025.pdf
```

**핵심**: 고객명별로 **자동 그룹화** → 한 화면에서 **일괄 매핑** → 한 번에 처리

---

#### Case D: 혼합 케이스 (자동 매칭 + 선택 필요)

```
[AR 8개 드롭]

  ✅ 자동 매칭 완료 (4개)
    - 홍길동 (2개) → 기존 고객 "홍길동" 정확히 1명 매칭
    - 김철수 (2개) → 기존 고객 "김철수" 정확히 1명 매칭

  ⚠️ 선택 필요 (4개)
    - 이영희 (2개) → "이영희A", "이영희B" 2명 매칭 → 선택 필요
    - 최신규 (2개) → 0명 매칭 → 새 고객 생성 또는 기존 고객 선택
```

**핵심**: 자동 처리 가능한 것은 처리하고, **사용자 개입이 필요한 그룹만 UI에 표시**

### 5.5 엣지 케이스 처리

#### 5.5.1 동일 고객명, 다른 발행일 중복

```
입력: AR_홍길동_2024.pdf (발행일: 2024.03.15)
기존: 홍길동 고객에게 이미 2024.03.15 발행 AR 존재

→ 해당 파일에 ⚠️ 표시
→ "이미 동일 발행일 AR이 존재합니다. 건너뛰시겠습니까?"
→ 옵션: [건너뛰기] [덮어쓰기]
```

#### 5.5.2 파일 해시 중복

```
입력: AR_홍길동_2024.pdf
기존: 동일 해시의 파일이 이미 등록됨

→ 해당 파일 자동 제외
→ 로그에 "중복 파일 제외: AR_홍길동_2024.pdf"
```

#### 5.5.3 AR 분석 실패

```
입력: 손상된_파일.pdf
결과: AR 분석 실패 (메타데이터 추출 불가)

→ 별도 "분석 실패" 그룹에 표시
→ 사용자가 수동으로 고객 지정 가능
```

#### 5.5.4 그룹 내 일부 파일만 중복

```
그룹: 홍길동 (3개 파일)
  - AR_홍길동_2024.pdf      ✅ 등록 가능
  - AR_홍길동_2025_01.pdf   ⚠️ 발행일 중복
  - AR_홍길동_2025_07.pdf   ✅ 등록 가능

→ 중복 파일은 체크박스로 제외 가능
→ "2개 파일 등록 예정 (1개 중복 제외)"
```

### 5.6 데이터 구조 설계

```typescript
// 분석 결과 그룹
interface ArFileGroup {
  /** AR에서 추출한 고객명 */
  customerNameFromAr: string;

  /** 이 그룹에 속한 AR 파일들 */
  files: ArFileInfo[];

  /** 매칭된 기존 고객 목록 */
  matchingCustomers: Customer[];

  /** 매칭 상태 */
  matchStatus: 'auto' | 'needs_selection' | 'no_match';

  /** 사용자가 선택한 고객 ID (null = 새 고객 생성) */
  selectedCustomerId: string | null;

  /** 새 고객 생성 시 사용할 이름 */
  newCustomerName?: string;
}

interface ArFileInfo {
  file: File;
  fileId: string;

  /** AR 메타데이터 */
  metadata: {
    customer_name: string;
    issue_date: string;
    report_title?: string;
  };

  /** 중복 상태 */
  duplicateStatus: {
    isHashDuplicate: boolean;
    isIssueDateDuplicate: boolean;
    existingDocId?: string;
  };

  /** 등록 포함 여부 (중복 시 사용자가 제외 가능) */
  included: boolean;
}

// 일괄 매핑 모달 상태
interface BatchMappingState {
  /** 전체 그룹 목록 */
  groups: ArFileGroup[];

  /** 모달 표시 여부 */
  isOpen: boolean;

  /** 모두 선택 완료 여부 */
  isAllSelected: boolean;

  /** 등록 진행 중 여부 */
  isProcessing: boolean;

  /** 진행률 (0~100) */
  progress: number;
}
```
