# Annual Report Phase 2 테스트 시나리오

## 테스트 환경 설정

### 1. 프론트엔드 개발 서버 시작

```bash
cd D:/aims/frontend/aims-uix3
PORT=3005 npm run dev
```

서버가 시작되면 브라우저에서 `http://localhost:3005` 접속

### 2. 백엔드 Python API 서버 확인

tars 서버에서 Python API가 실행 중인지 확인이 필요합니다.

**확인 방법:**
```bash
ssh rossi@tars.giize.com "ps aux | grep 'python.*8081'"
```

만약 실행 중이 아니면, tars에서 다음 명령으로 시작:
```bash
cd ~/aims/backend/api/doc_status_api
python main.py
```

## 테스트 시나리오

### 시나리오 1: 고객 1명 존재 (자동 선택 UI)

**목적:** 단일 고객이 검색될 때 자동 선택 확인 UI가 정상 동작하는지 확인

**절차:**
1. AIMS 좌측 메뉴에서 **"문서 등록"** 클릭
2. Annual Report PDF 파일 업로드 (예: `samples/pdf/` 폴더에 있는 파일)
3. 업로드 완료 후 자동으로 **"Annual Report 감지"** 모달이 나타나는지 확인

**확인 사항:**
- [ ] 모달 제목: "Annual Report 감지"
- [ ] 📄 파일명이 올바르게 표시됨
- [ ] 고객명 (PDF에서 추출된 이름)
- [ ] 발행일 (YYYY-MM-DD 형식)
- [ ] FSR 이름 (있는 경우)
- [ ] "다음 고객의 Annual Report로 자동 등록됩니다" 메시지
- [ ] 고객 정보 카드에 이름, 연락처 표시
- [ ] **"확인"** 버튼과 **"취소"** 버튼 표시

**액션:**
- **"확인"** 버튼 클릭
- 모달이 닫히는지 확인
- 브라우저 콘솔에 다음 로그 확인:
  ```
  [DocumentRegistrationView] Annual Report 파싱 요청 성공: {...}
  ```

---

### 시나리오 2: 동명이인 (2명 이상)

**목적:** 여러 명의 동명이인이 검색될 때 선택 UI가 정상 동작하는지 확인

**사전 준비:**
1. 같은 이름의 고객을 2명 이상 생성 (예: "김철수")
   - 고객 관리 → 신규 고객 생성
   - 동일한 이름, 다른 연락처로 2명 생성

**절차:**
1. 해당 고객명이 포함된 Annual Report PDF 업로드
2. 모달이 자동으로 나타나는지 확인

**확인 사항:**
- [ ] "김철수 이름으로 2명의 고객이 검색되었습니다" 메시지
- [ ] "해당하는 고객을 선택해주세요:" 안내 문구
- [ ] 라디오 버튼으로 고객 선택 UI 표시
- [ ] 각 고객의 정보 표시:
  - 이름
  - 전화번호 또는 이메일
- [ ] 선택 전에는 **"선택 완료"** 버튼이 비활성화됨
- [ ] 라디오 버튼 선택 시 배경색 변경 (선택된 항목 강조)

**액션:**
- 라디오 버튼으로 원하는 고객 선택
- **"선택 완료"** 버튼이 활성화되는지 확인
- **"선택 완료"** 버튼 클릭
- 모달이 닫히고 파싱 요청이 전송되는지 콘솔에서 확인

---

### 시나리오 3: 고객 없음

**목적:** 고객이 검색되지 않을 때 안내 메시지가 표시되는지 확인

**절차:**
1. 시스템에 등록되지 않은 이름의 Annual Report PDF 업로드
   - 예: "홍길동"이라는 이름이 PDF에 있지만 시스템에는 미등록
2. 모달이 자동으로 나타나는지 확인

**확인 사항:**
- [ ] 👤 아이콘 표시
- [ ] "이름으로 등록된 고객이 없습니다" 메시지
- [ ] "고객을 먼저 생성한 후 다시 업로드해주세요" 안내
- [ ] **"취소"** 버튼만 표시 (확인 버튼 없음)

**액션:**
- **"취소"** 버튼 클릭
- 모달이 닫히는지 확인
- 파싱 요청이 전송되지 않는지 확인 (콘솔에 로그 없음)

---

### 시나리오 4: 일반 PDF (Annual Report 아님)

**목적:** Annual Report가 아닌 일반 PDF는 모달이 나타나지 않는지 확인

**절차:**
1. 일반 PDF 파일 업로드
   - 보험청구서
   - 계약서
   - 기타 문서
2. 업로드 완료 대기

**확인 사항:**
- [ ] 모달이 **나타나지 않음**
- [ ] 일반 문서로 정상 처리됨
- [ ] 콘솔에 에러가 없음

---

### 시나리오 5: UI/UX 테스트

**목적:** 애플 디자인 철학이 제대로 적용되었는지 확인

**절차:**
1. 시나리오 1 또는 2를 실행하여 모달 표시
2. 다크 모드/라이트 모드 전환
3. 애니메이션 확인

**확인 사항:**
- [ ] **Progressive Disclosure:**
  - 처음에는 파일 정보만 표시
  - 스크롤하면서 단계적으로 정보 확인 가능
- [ ] **Subtle Interaction:**
  - 모달 등장 애니메이션 (fadeIn + slideUp)
  - 버튼 호버 시 배경색 변경이 부드러움
  - 라디오 버튼 선택 시 배경색 변경이 자연스러움
- [ ] **Light/Dark 테마:**
  - 테마 전환 버튼 클릭 (우측 상단)
  - 모달 배경색, 텍스트 색상이 자동으로 변경됨
  - 색상 대비가 충분하여 가독성이 좋음
- [ ] **반응형:**
  - 창 크기 조절 시 모달이 화면 중앙에 위치
  - 모달 너비가 화면의 90% 또는 최대 500px

## 디버깅 가이드

### 모달이 나타나지 않는 경우

**1. 백엔드 API 연결 확인:**
```bash
# 브라우저 콘솔에서 실행
fetch('http://tars.giize.com:8081/health')
  .then(r => r.json())
  .then(console.log)
```

**2. 콘솔 로그 확인:**
```
[DocumentRegistrationView] Annual Report 체크 실패: ...
```

**3. 네트워크 탭 확인:**
- `http://tars.giize.com:8081/annual-report/check` 요청 확인
- HTTP 상태 코드 확인 (200이어야 함)
- 응답 데이터의 `is_annual_report` 필드 확인

### API 호출 실패 시

**백엔드 서버 로그 확인:**
```bash
ssh rossi@tars.giize.com "tail -f ~/aims/backend/api/doc_status_api/logs/app.log"
```
