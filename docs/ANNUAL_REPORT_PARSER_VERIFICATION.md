## 발견된 오류 상세

### 오류 1: 신상철 계약 2 - 괄호 suffix 별도 라인

**원본 패턴:**
```
무배당 백만인을 위한 달러종신보험     <- 상품명 (윗줄에서 완결)
2 0013107410 신상철 신상철 ...       <- 메인 데이터
(저해지환급금형)                      <- suffix가 별도 라인
```

**문제**: 상품명이 윗줄에서 "보험"으로 끝나서 완결된 것으로 인식되어, 아랫줄의 `(저해지환급금형)` suffix를 병합하지 않음

**원인 분석**:
- 현재 파서는 윗줄이 불완전할 때만 (예: `(무해약환급` 으로 끝날 때) suffix를 병합
- 이 케이스는 윗줄이 완전한 상품명으로 끝나지만, 아랫줄에 별도로 suffix가 있는 패턴

**수정 필요**: 아랫줄이 `(`로 시작하는 괄호 suffix 패턴인 경우에도 상품명에 병합하도록 로직 추가

## 처리된 특수 패턴

1. **상품명 줄바꿈**: 윗줄 상품명 + 아랫줄 suffix (예: `Ⅱ보험`, `금형)`)
2. **계약자 줄바꿈**: 윗줄 계약자 첫부분 + 아랫줄 나머지 (예: `캐치업` + `코리아`)
3. **상태 줄바꿈**: 윗줄 상태 첫부분 + 아랫줄 나머지 (예: `업무처` + `리중`)
4. **Plus suffix**: 상품명에 ` Plus` 포함 (공백 유지)
5. **Plus(suffix) 복합**: `Plus(저해약환급금형)` 같은 복합 패턴
