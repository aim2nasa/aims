### BUG-03: `useDocumentStatistics` 비동기 정리 부족 [오탐]

**심각도**: ~~높음~~ 해당 없음
**유형**: 프론트엔드 Race Condition
**상태**: Alex 이론적 위험만 / Gini FAIL — **수정 불필요**
**파일**: `frontend/aims-uix3/src/hooks/useDocumentStatistics.ts`

#### 설명

`setInterval`이 `fetchStatistics()`를 비동기 호출하는데, 컴포넌트 언마운트 시 진행 중인 fetch를 `AbortController`로 취소하지 않음. `mountedRef`만으로 보호하고 있어 race condition 가능.

#### 관련 코드

```typescript
// 30초마다 통계 새로고침 (Freshness Guardian)
freshnessIntervalRef.current = setInterval(() => {
  fetchStatistics(true)  // 비동기 호출, cleanup 보장 없음
}, 30000)
```

#### 영향

- ~~컴포넌트 언마운트 후 상태 업데이트 시도~~
- ~~enabled 빠른 토글 시 stale data / 깜빡임~~

> **Alex 검증**: `mountedRef.current` 가드가 fetch 완료 시점에서 setState를 보호함. `AbortController` 미사용은 불필요한 네트워크 요청이 완료될 때까지 유지되는 것일 뿐, 기능적 버그가 아님. 테스트도 `enabled true→false→true` 전환까지 커버.
> **Gini 검증**: `mountedRef` 패턴은 React 공식 권고 패턴. cleanup에서 `setInterval` 정리도 올바르게 구현됨.

### BUG-04: SSE 구독 복잡한 의존성 [오탐]

**심각도**: ~~중간~~ 해당 없음
**유형**: 프론트엔드 Race Condition
**상태**: Alex 올바른 패턴 / Gini FAIL — **수정 불필요**
**파일**: `frontend/aims-uix3/src/shared/hooks/useSSESubscription.ts`

#### 설명

여러 콜백 ref 업데이트와 SSE 재연결 간 경쟁 조건 가능. `connect` 함수가 의존성에 포함되면서 ref들을 사용하기 때문에 클로저 문제 발생 가능성.

#### 영향

- ~~탭 활성화/비활성화 시 토큰 동기화 누락 가능~~
- ~~중복 구독 발생 가능~~

> **Alex 검증**: ref 패턴이 stale closure 방지를 위한 정석 패턴. `connect`는 `[enabled, streamKey]`에만 의존하고 나머지는 ref로 최신 값 참조. cleanup에서 모든 리소스(구독, 이벤트 리스너) 완전 정리됨.
> **Gini 검증**: React best practice를 따르는 올바른 구현. 중복 구독은 cleanup에서 방지됨.

### BUG-05: `annualReportApi` `as any` 타입 캐스팅 [코드 품질]

**심각도**: 낮음 (기술 부채)
**유형**: 타입 안전성
**상태**: Alex 의도된 설계 / Gini PASS — **P2 개선 권장**
**파일**: `frontend/aims-uix3/src/features/customer/api/annualReportApi.ts` (Line 109-131)

#### 설명

```typescript
const contractData = contract as any
const policyNumber = contractData['증권번호'] || contractData.contract_number
```

서버 응답의 계약 데이터가 **한글 키**(`'증권번호'`, `'보험사'`)와 **영문 키**(`contract_number`)로 올 수 있어 `as any` 사용. `||` fallback으로 런타임 에러는 방지됨.

> **Alex 검증**: 의도된 트레이드오프. DB의 실제 필드가 한글 키(`증권번호`, `보험상품`, `계약상태` 등)이고, `InsuranceContract` 인터페이스는 영문만 정의. 양쪽 대응을 위한 합리적 방어적 프로그래밍. `!policyNumber` 시 `continue`로 안전 스킵.
> **Gini 검증**: MongoDB에서 실제 데이터 확인. 한글 필드명을 포함한 Union 타입 정의 시 `as any` 제거 가능. 현재 동작에는 문제 없음.
