## 3. FAIL 분석 — 버그 없음

### 3.1 최소 포함 4건 FAIL — 범용 키워드 경쟁

| 쿼리 | 결과 건수 | 상황 |
|------|:---------:|------|
| "보험" | 20건 | 거의 모든 문서에 "보험" 포함 → 특정 문서가 21위 이하로 밀림 |
| "보장" | 20건 | 동일 |
| "계약" | 20건 | 동일 |
| "보험금" | 20건 | 동일 |

반면 **구체적 키워드는 모두 PASS**:
```
'곽지민' → 14건 반환, 기대 문서 포함 ✅
'김성민' → 2건 반환, 기대 문서 포함 ✅
'김가원' → 1건 반환, 기대 문서 포함 ✅
```

**판단**: 범용 키워드에서 수백 건이 경쟁하여 상위 20건에 못 든 것. 검색 로직 문제가 아닌 테스트 설계 한계.

### 3.2 교차 검증 4건 FAIL — 랭킹 기준 차이

```
'보험' → 키워드 20건 vs AI 20건 = 겹침 0건

'진단서' → 겹침 0건
'보장' → 겹침 0건
```

키워드 검색은 **파일명 매칭 가중치 3점**, AI 검색은 **벡터 유사도 0.89** 기준으로 랭킹하므로 상위 결과가 완전히 다를 수 있음. 이는 **두 검색 방식의 근본적 특성 차이**이며 버그가 아님.
