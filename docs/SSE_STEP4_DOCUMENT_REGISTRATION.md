# 4단계: 문서 등록 폴링 → SSE 전환

## 개요

문서 등록 후 백엔드 처리 완료를 감지하는 방식을 **5초 폴링**에서 **SSE(Server-Sent Events)**로 전환.

---

## 핵심 (본질)

**"문서 처리가 끝났는지"만 알면 됨. OCR 여부와 무관.**

```
업로드 완료
    │
    ▼
┌─────────────────────────────────┐
│  백엔드 처리 중...               │
│  (텍스트 추출, 메타데이터 등)     │
└────────────────┬────────────────┘
                 │
     ┌───────────┴───────────┐
     │                       │
     ▼                       ▼
 OCR 필요               OCR 불필요
 (텍스트 없음)           (텍스트 있음)
     │                       │
     ▼                       ▼
 OCR 처리              바로 완료
     │                       │
     └───────────┬───────────┘
                 │
                 ▼
         ┌─────────────┐
         │   NOTIFY    │  ← 이것만 있으면 됨
         └─────────────┘
```

## 구현 시 발견된 이슈

### n8n 표현식 따옴표 문제

**문제**: n8n 표현식에서 `$('Pasre').item.json.doc_id`로 전달하면 값에 따옴표가 포함됨
```
n8n 전송: "6944e3e1ed26f7226e932414" (26자, 따옴표 포함)
SSE Map 키: 6944e3e1ed26f7226e932414 (24자, 따옴표 없음)
```

**해결**: server.js webhook에서 따옴표 제거
```javascript
const documentIdStr = document_id.toString().replace(/^"|"$/g, '');
notifyDocumentStatusSubscribers(documentIdStr, 'processing-complete', eventData);
```
