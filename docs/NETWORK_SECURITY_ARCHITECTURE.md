# AIMS 네트워크 보안 아키텍처

## 개요

AIMS는 Tailscale VPN을 통한 보안 네트워크 구조를 채택하여 백엔드 서버에 대한 직접 접근을 차단하고, 인증된 클라이언트만 접속할 수 있도록 합니다.

## 보안 정책

### 1. 외부 접근 차단
- 백엔드 포트 (3010, 8000, 3011, 27017)는 **UFW 방화벽**으로 외부 접근 차단
- 공인 IP (tars.giize.com)로는 백엔드 서비스에 직접 접근 불가

### 2. Tailscale VPN 필수
- 모든 개발 환경에서 백엔드 접속 시 **Tailscale VPN** 사용 필수
- Tailscale IP: `100.110.215.65` (tars 서버)
- 인증된 Tailscale 계정만 네트워크 참여 가능

### 3. 프론트엔드 프록시 설정
```typescript
// vite.config.ts
proxy: {
  '/api': {
    target: 'http://100.110.215.65:3010',  // Tailscale VPN (보안 접속)
    secure: false,
    changeOrigin: true
  }
}
```

## 장점

| 항목 | 설명 |
|------|------|
| 암호화 | 모든 트래픽이 Tailscale WireGuard 터널로 암호화 |
| 인증 | Tailscale 계정 인증 필수 (무단 접근 불가) |
| 방화벽 우회 | 공유기/방화벽 설정 없이도 안전한 접속 |
| 감사 추적 | Tailscale 관리 콘솔에서 접속 기록 확인 가능 |

## 프로덕션 환경

프로덕션 배포 시에는 nginx HTTPS 프록시 (aims.giize.com)를 통해 접속:
- 외부 사용자: `https://aims.giize.com` → nginx → 내부 서비스
- 개발자: Tailscale VPN → 직접 접속 (디버깅/개발용)
