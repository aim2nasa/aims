# AI 어시스턴트 응답 품질 개선

> **목표**: AI 어시스턴트가 상세문서검색과 동등한 수준의 RAG 응답 품질을 제공
> **시작일**: 2026-03-16
> **상태**: 🔄 진행 중

---

## 1. 문제

### 동일 질문, 다른 결과

**질문**: "캐치업코리아 자동차 정보 알려줘"

| | 상세문서검색 (질문 검색) | AI 어시스턴트 |
|---|---|---|
| AI 답변 | BMW iX xDrive 50, 차량번호, 보험기간, 차량가액 등 **정확한 자동차 정보** | "건강검진 문서"가 1번, 자동차 문서는 3~5번 |
| 검색 결과 | 312건, 유사도 정렬 | 키워드 267건 + AI 50건, **정렬 없음** |
| 검색 경로 | `tars.giize.com/search_api` → nginx → `localhost:8000/search` (semantic) | `unified_search` → `localhost:8000/search` (keyword + semantic) |

**같은 RAG API를 사용하는데 응답 품질이 현저히 다름 — 용납 불가**

---

## 2. 원인 분석 (Alex/Gini 합의)

### 원인 1 (Major): SmartSearch 키워드 검색에 랭킹 없음
- `smart_search.py:129` — MongoDB `find()` 결과를 삽입 순서로 반환
- "캐치업코리아"+"자동차" 2개 매칭 문서와 "정보" 1개 매칭 문서가 동등 취급
- **건강검진 문서가 1번인 이유: 단순히 먼저 DB에 저장되었기 때문**

### 원인 2 (Major): 불용어(stop word) 미제거
- `smart_search.py:99` — 쿼리를 공백 분리만 수행
- "캐치업코리아 자동차 정보 알려줘" → `["캐치업코리아", "자동차", "정보", "알려줘"]`
- "정보", "알려줘" 같은 불용어가 무관한 문서를 대량 매칭

### 원인 3 (Minor): 폴백 쿼리 미정제
- `chatService.js` — 사용자 원문 전체를 unified_search query로 전달
- "알려줘" 같은 불용어가 포함된 채 검색

### 비교: 상세문서검색은 왜 잘 되는가?
- 상세문서검색은 **semantic 모드**만 사용 → Qdrant 벡터 검색 → 유사도 점수 기반 정렬
- AI 어시스턴트의 unified_search는 **keyword + semantic 병행** → 키워드 결과가 앞에 배치 → 랭킹 없는 키워드 결과가 응답을 오염

## 3. 수정 계획

### 수정 1: SmartSearch 불용어 필터링 (즉시)
- `smart_search.py`에 한국어 불용어 리스트 추가
- "정보", "알려줘", "보여줘", "해줘", "있어", "없어" 등 제거

### 수정 2: SmartSearch 키워드 매칭 점수 기반 정렬 (즉시)
- 매칭된 키워드 수가 많은 문서를 상위로 정렬
- "캐치업코리아"+"자동차" 2개 매칭 > "정보" 1개 매칭

### 수정 3: 폴백 쿼리 정제 (보조)
- `chatService.js`에서 사용자 원문 대신 의미 있는 키워드만 추출하여 전달
