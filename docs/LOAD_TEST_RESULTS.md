|------|-----|
| 테스트 일시 | 2026-01-05 05:35~05:42 KST |
| 대상 서버 | https://aims.giize.com |
| 테스트 도구 | Node.js 기반 자체 스크립트 |
| 테스트 방식 | **200개 독립 사용자 계정**, 파일 업로드 포함 |

## 테스트 시나리오

각 가상 사용자(VU)는 독립적인 JWT 토큰으로 다음 동작을 반복:
1. 문서 목록 조회 (`GET /api/documents`)
2. 0.5~1.5초 대기
3. 고객 목록 조회 (`GET /api/customers`)
4. 0.5~1.5초 대기
5. 검색 (30% 확률) (`GET /api/documents?search=...`)
6. 파일 업로드 (10% 확률) (`POST /shadow/docprep-main`)
7. 1~3초 대기 후 반복

파일 업로드 응답시간이 긴 이유:
1. **n8n 워크플로우 경유**: shadow/docprep-main 엔드포인트

3. **DB 저장**: MongoDB 문서 생성
