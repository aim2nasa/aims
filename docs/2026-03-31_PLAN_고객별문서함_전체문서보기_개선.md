## 1. 배경

문서 업로드 후 최근 파일을 찾기 어려운 문제. 고객별 문서함은 트리 구조(문서유형별 분류)라서, 업로드된 파일이 여러 카테고리에 흩어져 위치 파악이 어렵다.

## 2. 설계 원칙

- **페이지별 역할 분리**: 각 페이지의 본질적 목적을 혼재하지 않는다
- **고객별 문서함**: 특정 고객의 문서를 유형별로 탐색 (분류 탐색이 본질)
- **전체 문서 보기**: 전체 문서의 처리 상태 모니터링 + 날짜순 탐색 (상태 모니터링이 본질)

## 3. 에이전트 회의 결과 (Dana/Sora/Alex/Merlin)

### 핵심 결론
- 소규모 업로드(4~20건): 고객별 문서함 내 하이라이트로 해결
- 대규모 업로드(수백건): 전체 문서 보기가 담당 (역할에 맞음)
- 고객별 문서함에 날짜순 리스트를 넣는 것은 전체문서보기와 기능 중복 → 하지 않음

### Sora(보험 설계사) 핵심 피드백
- 파일 올리고 "다 들어간 거 맞아?" 확인이 제일 불안함
- 폴더 접혀 있어도 "이 안에 새 파일 있어요" 표시 필수
- 고객 앞에서 3초 안에 찾아야 함
- 필터 개념은 50~60대에게 위험 (켜놓고 까먹음)

## 4. 개선 항목

### A. 고객별 문서함 (3건)

#### A-1. 카테고리 NEW 배지
- **내용**: 오늘 등록된 문서가 있는 대분류/소분류 폴더명 옆에 숫자 배지 표시
- **형태**: 작은 파란 점 + 숫자 (예: `보험금청구 (64건) ·2`)
- **표시 조건**: 오늘(KST) 등록된 문서가 해당 카테고리에 있을 때만
- **접힌 상태 가치**: 트리가 접혀 있어도 "이 카테고리에 새 문서가 있구나" 인지 가능

#### A-2. 문서 행 하이라이트
- **내용**: 오늘 등록된 문서 행에 좌측 2px 파란 세로선 (accent color)
- **소멸 조건**: 세션 내 유지, 새로고침 또는 30분 경과 시 자연스럽게 사라짐
- **효과**: 폴더 펼쳤을 때 "이게 오늘 올린 파일" 3초 안에 파악

#### A-3. 상태바 → 전체문서보기 점프 링크
- **내용**: ExplorerProcessingStatusBar의 "에러 N건", "처리중 N건" 텍스트를 클릭 가능 링크로 변경
- **동작**: 클릭 시 전체문서보기로 이동 + 해당 상태 필터 + 해당 고객 필터 자동 적용
- **효과**: 대규모 업로드 시 상세 상태 확인을 전체문서보기로 자연스럽게 위임

### B. 전체 문서 보기 (2건)

#### B-1. 고객 필터 프로덕션 개방
- **현황**: `isDevServer` 조건(`window.location.hostname === 'localhost'`)으로 숨겨져 있음
- **변경**: 조건 제거하여 프로덕션에서도 고객별 필터 사용 가능
- **사전 확인**: isDevServer로 숨긴 이유 파악 (성능/UX 문제 이력 확인)

#### B-2. 처리 상태 필터 탭
- **내용**: 전체/처리중/완료/에러 세그먼트 컨트롤 추가
- **현황**: documents/status API가 이미 status 파라미터 지원. 프론트엔드 UI만 추가
- **효과**: "에러난 것만 보기", "처리중인 것만 보기" 가능
