## 개선 이슈 (향후)

### 1. HWP → PDF 변환 성능 (HIGH)

**현상**: HWP 파일당 변환에 약 60초 소요 (hwp5odt ~30초 + LibreOffice ~30초)
**영향**: 3건 순차 처리 시 총 3분+ 대기. 사용자 UX 저하.
**원인**:
- hwp5odt (Python 기반 HWP → ODT 변환)가 본질적으로 느림
- LibreOffice 동시 실행 불가로 `concurrency=1` 제한

**개선 방안 후보**:
- A. LibreOffice 인스턴스 풀링 (복수 포트로 병렬 처리)
- B. HWP 변환 전용 워커 분리 + 별도 큐
- C. hwp5html 등 경량 변환기 검토
- D. 변환 중 상태를 UI에 실시간 표시 (progress: "변환 중 1/3")

### 2. 일괄등록 시 파일명에 폴더 prefix 포함 (MEDIUM)

**현상**: 문서 일괄등록으로 업로드 시 `캐치업코리아/표준취업규칙(최종).hwp`처럼 폴더명이 파일명에 포함
**영향**: 전체 문서 보기에서 파일명이 길어지고 폴더명이 반복 표시
**원인**: `upload.originalName`에 폴더 경로가 포함되어 저장됨

**개선 방안**: 일괄등록 시 폴더 prefix를 제거하여 파일명만 저장 (폴더명은 별도 필드에 보관)

### 3. 동일 내용 HWP의 displayName 중복 (LOW)

**현상**: 내용이 유사한 HWP 3건이 모두 `✦캐치업코리아 취업규칙(최종).hwp`로 동일한 displayName 생성
**영향**: 문서 목록에서 구분 불가
**원인**: AI가 문서 내용 기반으로 제목을 생성하므로, 유사한 내용 → 유사한 제목

**개선 방안**: displayName 중복 시 날짜/크기/원본 파일명 기반 suffix 추가

### 4. 보관 파일의 문서유형 "미지정" 표시 (LOW)

**현상**: ZIP, AI, JPG 등 텍스트 추출 불가 파일이 "미지정"으로 표시
**영향**: 사용자가 처리 실패로 오해할 수 있음
**동작**: 현재는 정상 동작 (텍스트 없으면 AI 분류 불가)

**개선 방안**: "미지정" 대신 "보관" 또는 파일 확장자 기반 기본 유형 부여 (예: ZIP → "압축파일")

### 5. 텍스트 없는 파일의 파일명 기반 분류 (MEDIUM)

**현상**: ZIP, AI, JPG 등 텍스트 추출 불가 파일은 AI 분류를 수행하지 않아 "미지정"으로 남음
**영향**: 파일명에 유의미한 정보가 있어도 분류되지 않음 (예: "고객거래확인서.zip" → 미지정)
**원인**: 현재 분류 로직이 `full_text`가 10자 이상일 때만 AI 분류 호출

**개선 방안**: 텍스트 추출 불가 파일은 파일명(originalName)을 기반으로 AI 분류 수행
- 예: `캐치업코리아노무규정.zip` → "인사/노무"
- 예: `고객거래확인서,FATCA확인서.zip` → "일반문서" 또는 "개인서류"
- `summarize_text()` 대신 파일명 전용 분류 프롬프트 사용

### 6. 변환 중 상태 표시 개선 (MEDIUM)

**현상**: HWP 업로드 직후 "완료 + conversion_failed"로 표시되다가 3분 후 갑자기 "TXT + 인사/노무"로 변경
**영향**: 사용자가 변환 실패로 오해 → 재시도 → 불필요한 중복 업로드 가능
**원인**: 파이프라인이 변환 완료를 기다리지 않고 먼저 "완료" 상태로 마킹

**개선 방안**:
- 변환 대기 중인 문서에 "PDF 변환 중..." 상태 표시
- SSE를 통해 변환 완료 시 자동 UI 갱신
