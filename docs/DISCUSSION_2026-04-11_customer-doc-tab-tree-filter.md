# 고객 상세 문서 탭 — 분류 필터 트리 네비게이션 전환

- **일자**: 2026-04-11
- **작업 유형**: feature 수정 개발 (소규모 UI 개선)
- **프로세스**: Compact Fix (`/compact-fix`)
- **브랜치**: `feat/customer-doc-tab-tree-filter`

---

## 1. 배경

고객 상세 화면(`CustomerDetailView`)의 **문서 탭**은 분류 필터로 단순 드롭다운을 사용한다. 드롭다운은 6개 최상위 대분류만 평면 노출하므로, 사용자가 특정 소분류(예: "보험증권")만 골라 보는 것이 불가능하다.

이미 시스템에는 **3단 계층(대분류 → 소분류 → 파일)**을 보여주는 트리 빌더가 존재한다 (`고객별 문서함` / `documents-explorer` 화면). 다만 이 화면은 별도 메뉴 진입점이며 본 작업의 수정 대상이 아니다 — 사용자에게는 참고용으로만 제시되었다.

---

## 2. 사용자 요구사항

좁은 공간(고객 상세 문서 탭)에 가급적 많은 정보를 논리적으로 보여주기 위해, **드롭다운을 폴더 트리 네비게이션으로 교체한다.**

### 사용자 예시 (원문 인용)

> "보험계약 폴더를 선택했다면, 아래 목록에는 '보험증권' '청약서' '가입설계서' '기타 보험관련' 폴더들이 폴더 아이콘을 갖고 보여주면 되고, 보험증권 폴더를 선택했다면, 이 안에 들어 있는 모든 파일들을 보녀주면 되는거야."

---

## 3. 합의된 결정 사항

| # | 항목 | 결정 | 이유 |
|---|------|------|------|
| 1 | 네비게이션 방식 | **드릴다운(Finder/탐색기 스타일)** — 한 화면에 한 단계 | 좁은 공간에 적합. 트리 펼침/접기는 답답함 |
| 2 | "전체" 옵션 | **없음** — 루트엔 6개 대분류 폴더만 | 트리 네비게이션 본질에 충실. 단순화 |
| 3 | 표시 모드 | **폴더만 OR 파일만** (한 레벨 = 한 종류) | 혼재 시 사용자 혼란. 명확성 우선 |
| 4 | 빈 폴더 처리 | **숨김** (하위에 자식이 없으면 표시 안 함) | 노이즈 제거 |
| 5 | 수정 범위 | 고객 상세 → 문서 탭의 분류 드롭다운만 | 지정된 영역 외 절대 변경 금지 |

### 트리 계층 구조

```
[루트: "분류"]
├─ 🛡️ 보험계약 (164)        ← 대분류 (6개 중 하위 자식 있는 것만)
│   ├─ 📁 보험증권 (125)     ← 소분류 (22 doc types 중 해당 대분류 소속)
│   ├─ 📁 청약서 (9)
│   ├─ 📁 가입설계서 (21)
│   └─ 📁 기타 보험관련 (9)
├─ 🏥 보험금청구 (31)
│   └─ ...
├─ 🪪 신분/증명 (18)
│   └─ ...
└─ ...
```

소분류 클릭 시 → 그 소분류에 속하는 **파일 목록**으로 전환.

## 4. 수정 범위

### 변경 대상

| 파일 | 변경 내용 |
|------|-----------|
| `frontend/aims-uix3/src/features/customer/views/CustomerDetailView/tabs/DocumentCategoryFilter.tsx` | 드롭다운 → 드릴다운 트리 네비게이터로 교체. 브레드크럼 + 폴더 그리드 + 상위 이동 |
| `frontend/aims-uix3/src/features/customer/views/CustomerDetailView/tabs/DocumentsTab.tsx` | 필터 상태에 소분류(`subType`) 차원 추가. 새 컴포넌트 props 전달 |

### 변경 없는 영역 (절대 건드리지 않음)

- `CustomerDocumentExplorerView.tsx` ("고객별 문서함" 메뉴 — 참고용 화면)
- `DocumentExplorerTree.tsx`, `treeBuilders.ts` (재사용만 — 매핑 로직 import)
- `DOCUMENT_CATEGORIES` 상수 (재사용만)
- 백엔드 (변경 없음 — Phase 2.5 스킵)

### 재사용

- `DOCUMENT_CATEGORIES` (대분류 6개)
- `getCategoryForType()` (doc_type → 대분류 매핑)
- `getDocumentTypeLabel()` (doc_type → 한글 라벨)
- `getTypeDisplayOrder()` (소분류 정렬 순서)

## 5. UX 흐름

```
초기 상태
┌─ [분류]                                  (브레드크럼)
├─────────────────────────────────────
│  🛡️ 보험계약 (164)
│  🏥 보험금청구 (31)
│  🪪 신분/증명 (18)
│  🏢 자산 (3)
│  🏛️ 법인 (45)
│  📄 기타 (6)
└─────────────────────────────────────

"보험계약" 클릭
┌─ [분류 ▸ 보험계약]                       [← 상위로]
├─────────────────────────────────────
│  📁 보험증권 (125)
│  📁 청약서 (9)
│  📁 가입설계서 (21)
│  📁 기타 보험관련 (9)
└─────────────────────────────────────

"보험증권" 클릭
┌─ [분류 ▸ 보험계약 ▸ 보험증권]            [← 상위로]
├─────────────────────────────────────
│  (문서 파일 목록 — 기존 테이블 그대로)
│  📄 마리치_보험증권_2024.pdf
│  📄 ...
└─────────────────────────────────────
```

- 브레드크럼의 각 노드는 클릭 가능 (해당 레벨로 점프)
- "← 상위로"는 한 단계 위로
- 빈 폴더는 노출 안 함

## 6. 검증 계획 (Phase 3)

| 시나리오 | 기대 동작 |
|---------|----------|
| 마리치 고객 진입 | 트리 루트 — 6개 대분류 폴더 (count > 0인 것만) |

| "← 상위로" | 한 단계 위로 |
| 빈 분류 (모든 카운트 0인 고객) | 안내 메시지 |
| 파일명 검색 | 현재 위치 무관, 매칭 결과 표시 (기존 동작 유지) |
