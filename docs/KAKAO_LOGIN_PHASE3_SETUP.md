# Phase 3: 카카오 개발자 콘솔 설정 가이드

## 개요

이 문서는 AIMS 카카오 소셜 로그인의 Phase 3 설정 가이드입니다.

- **Phase 1**: 백엔드 인증 시스템 구현 ✅
- **Phase 2**: 프론트엔드 UI 구현 ✅
- **Phase 3**: 카카오 개발자 콘솔 설정 + 전체 통합 ⏳

---

## 1. 카카오 개발자 사이트 접속

1. https://developers.kakao.com/ 접속
2. 카카오 계정으로 로그인
3. "내 애플리케이션" 메뉴 선택

---

## 2. 애플리케이션 추가

1. **"애플리케이션 추가하기"** 클릭
2. 다음 정보 입력:
   - **앱 이름**: AIMS
   - **사업자명**: 회사명 입력
3. **"저장"** 클릭

---

## 3. 앱 키 확인

1. 생성된 앱 선택
2. **"앱 설정"** > **"요약 정보"** 메뉴
3. 다음 키 확인 및 복사:

| 키 종류 | 용도 | 사용 여부 |
|---------|------|-----------|
| 네이티브 앱 키 | iOS/Android 앱 | ❌ 사용 안 함 |
| **REST API 키** | 백엔드 서버 | ✅ **이것을 사용** |
| JavaScript 키 | 웹 프론트엔드 직접 호출 | ❌ 사용 안 함 |
| Admin 키 | 관리자 기능 | ❌ 사용 안 함 |

> ⚠️ **중요**: REST API 키를 복사하세요! JavaScript 키가 아닙니다!

---

## 4. 카카오 로그인 활성화

1. **"제품 설정"** > **"카카오 로그인"** 메뉴
2. **"활성화 설정"** 섹션에서 **"카카오 로그인"** ON으로 변경

---

## 5. Redirect URI 설정 (중요!)

1. **"제품 설정"** > **"카카오 로그인"** 메뉴
2. **"Redirect URI"** 섹션에서 **"Redirect URI 등록"** 클릭
3. 다음 URI 모두 추가:

```
http://localhost:3010/api/auth/kakao/callback
http://tars.giize.com:3010/api/auth/kakao/callback
```

> ⚠️ **주의사항**:
> - URI는 정확히 일치해야 합니다 (대소문자, 슬래시 포함)
> - 끝에 슬래시(/)가 없어야 합니다
> - http와 https를 구분합니다

---

## 6. 동의 항목 설정

1. **"제품 설정"** > **"카카오 로그인"** > **"동의 항목"** 메뉴
2. 다음 항목 설정:

| 항목 | 설정 | 설명 |
|------|------|------|
| **닉네임** | 필수 동의 | 사용자 이름 표시용 |
| **프로필 사진** | 선택 동의 | 아바타 이미지용 |
| **카카오계정(이메일)** | 선택 동의 | 이메일 표시용 |

> 💡 **참고**: "카카오계정(이메일)"을 사용하려면 "비즈 앱" 전환이 필요할 수 있습니다.

---

## 7. Client Secret 발급 (권장)

1. **"제품 설정"** > **"카카오 로그인"** > **"보안"** 메뉴
2. **"Client Secret"** 섹션에서 **"코드 생성"** 클릭
3. 생성된 Client Secret 복사
4. **"활성화 상태"**를 **"사용함"**으로 변경

> 💡 Client Secret은 보안 강화를 위해 사용하는 것을 권장합니다.

---

## 8. 백엔드 환경변수 설정

카카오 개발자 콘솔 설정이 완료되면, 백엔드 서버의 `.env` 파일을 업데이트합니다.

### 8.1. tars 서버 접속

```bash
ssh tars.giize.com
```

### 8.2. .env 파일 수정

```bash
cd ~/aims/backend/api/aims_api
nano .env
```

### 8.3. 다음 값 업데이트

```env
# 카카오 OAuth 설정
KAKAO_CLIENT_ID=여기에_REST_API_키_입력
KAKAO_CLIENT_SECRET=여기에_Client_Secret_입력
KAKAO_CALLBACK_URL=http://tars.giize.com:3010/api/auth/kakao/callback

# JWT 설정 (이미 설정되어 있다면 그대로 유지)
JWT_SECRET=your_jwt_secret_key_change_in_production
JWT_EXPIRES_IN=7d

# 프론트엔드 URL
FRONTEND_URL=http://localhost:5176
```

### 8.4. 서버 재시작

```bash
cd ~/aims/backend/api/aims_api
./deploy_aims_api.sh
```

## 9. 전체 로그인 플로우 테스트

### 9.1. 테스트 순서

1. 브라우저에서 `http://localhost:5176/login` 접속
2. **"카카오 로그인"** 버튼 클릭
3. 카카오 로그인 페이지로 리다이렉트 확인
4. 카카오 계정으로 로그인
5. 동의 화면에서 동의 항목 확인 후 동의
6. AIMS 메인 페이지로 리다이렉트 확인
7. 로그인 상태 유지 확인 (페이지 새로고침 후에도 로그인 유지)

## 10. 문제 해결

### 10.1. KOE101 에러 (앱 관리자 설정 오류)

**원인**: 카카오 로그인이 활성화되지 않았거나 Redirect URI가 설정되지 않음

**해결**:
1. 카카오 로그인 활성화 확인
2. Redirect URI가 정확히 등록되었는지 확인

### 10.2. KOE006 에러 (Redirect URI 불일치)

**원인**: 등록된 Redirect URI와 실제 요청 URI가 다름

**해결**:
1. 카카오 개발자 콘솔에서 Redirect URI 확인
2. 백엔드 `.env`의 `KAKAO_CALLBACK_URL`과 일치하는지 확인
3. 끝에 슬래시(/) 유무 확인

### 10.3. 토큰 발급 실패

**원인**: Client ID 또는 Client Secret이 잘못됨

**해결**:
1. REST API 키가 맞는지 확인 (JavaScript 키 아님)
2. Client Secret이 맞는지 확인
3. Client Secret 활성화 상태 확인

### 10.4. 사용자 정보 조회 실패

**원인**: 동의 항목이 설정되지 않음

**해결**:
1. 카카오 개발자 콘솔에서 동의 항목 설정 확인
2. 필수 동의 항목(닉네임) 확인

## 12. 보안 주의사항

1. **REST API 키와 Client Secret은 절대 프론트엔드 코드에 노출하지 마세요**
2. **`.env` 파일은 절대 Git에 커밋하지 마세요**
3. **프로덕션 환경에서는 HTTPS를 사용하세요**
4. **JWT Secret은 충분히 복잡한 값을 사용하세요**
