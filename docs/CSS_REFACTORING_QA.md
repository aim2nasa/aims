# CSS 구조 개선 Q&A (Claude Code 참고 자료)

---

## Q1. UI 레이아웃을 변경하려 하는데, CSS 규칙이나 알 수 없는 스타일 충돌로 제대로 변경이 안 된다. 근본적인 해결 방법은?

### 핵심 원인 진단

AIMS 프론트엔드의 CSS가 여러 곳에 분산되어 있어, 어떤 규칙이 어디서 적용되는지 추적이 어렵다. 컴포넌트별 CSS, 글로벌 CSS, variables.css, 인라인 스타일 등이 뒤섞이면 React 컴포넌트가 중첩될수록 CSS 우선순위(specificity) 문제가 복잡해진다.

### 근본 해결 방향 3가지

#### 방향 1: Tailwind CSS 도입 (전면 교체)

유틸리티 클래스 기반이라 CSS 파일 자체가 거의 필요 없어지고, 컴포넌트 JSX에서 바로 스타일을 확인·수정할 수 있다. "어디서 이 스타일이 오는 거지?" 문제가 원천 차단된다. 다만 기존 코드 전면 마이그레이션이 필요해서 작업량이 크다.

#### 방향 2: CSS Modules 점진 적용

이미 React + Vite 환경이므로 `.module.css`를 쓰면 클래스명이 자동으로 스코프되어 컴포넌트 간 스타일 충돌이 구조적으로 불가능해진다. 기존 CSS를 점진적으로 마이그레이션할 수 있어서 현실적이다.

#### 방향 3: 현재 CSS 시스템 정리 (리팩토링)

가장 보수적인 접근으로, Chrome DevTools의 Computed 탭에서 문제 요소의 스타일 상속 체인을 역추적하고, 불필요한 글로벌 규칙을 제거하면서 `variables.css` + 컴포넌트 CSS의 책임을 명확히 분리한다.

## Q2. 이 CSS 구조를 어떻게 개선하라고 해야 하는가?

### 현재 문제의 본질

CLAUDE.md에 CSS 규칙이 있지만, 그것은 "어떻게 써라"만 정의하고 **"전체 구조를 어떻게 정리해라"는 없는 상태**이다. 그래서 매번 기존 스타일과 충돌하면서 땜질식 수정만 반복하게 된다.

근본적인 해결은 **CSS 아키텍처 리팩토링 계획을 세우고, 그것을 CLAUDE.md나 별도 스킬로 등록하는 것**이다.

### 3단계 접근법

#### 1단계: 현재 CSS 구조 감사 (Audit)

```
frontend/aims-uix3/src 폴더에서 CSS 파일 전체 목록과 각 파일의 역할을 분석해줘.
글로벌 CSS, 컴포넌트 CSS, variables.css 간의 의존관계와 충돌 가능성을
보고서로 만들어줘.
```

#### 2단계: 문제 패턴 식별

감사 결과에서 보통 다음 문제들이 발견된다:

- 글로벌 CSS에서 태그 셀렉터(`div`, `span`)로 스타일 지정 → 모든 컴포넌트에 영향
- 같은 클래스명이 여러 파일에서 중복 정의
- specificity 전쟁 (클래스 vs ID vs 중첩 셀렉터)
- 레이아웃(flex/grid)과 장식(색상/폰트)이 한 파일에 혼재

#### 3단계: 구조 개선 지시

감사 결과를 기반으로 다음과 같이 지시한다:

```
CSS 감사 결과를 바탕으로 CSS 리팩토링 계획을 세워줘.
CSS Modules로 점진적 마이그레이션하는 방안을 제안하고,
레이아웃 변경이 쉬워지도록 레이아웃 CSS와 테마 CSS를 분리해줘.
완료되면 .claude/skills/css-architecture/SKILL.md로 새 규칙을 정리해줘.
```

### 기대 효과

이 과정을 거치면 일회성 수정이 아니라 **구조적으로 개선**되며, 이후 레이아웃 변경 작업도 훨씬 수월해진다. 특히 `.claude/skills/`에 규칙이 등록되면 이후 모든 CSS 작업에서 자동으로 적용된다.
