# 문서 업로드 진행률 멈춤 버그 해결

> 해결일: 2026-01-19

---

## 문제

문서 업로드 중 다른 페이지 갔다 오면 진행률이 20%/40%에서 멈춰 보임.
F5 새로고침해야 100% 완료로 표시됨.

---

## 원인

```
1. 문서 목록 페이지에서 업로드 시작 (20% 표시)
2. 다른 페이지로 이동 → SSE 연결 끊김
3. 백엔드가 40%, 60%, 80%, 100% 이벤트 보냄 → 받을 곳이 없음 (손실)
4. 문서 목록 페이지로 복귀 → 마지막으로 받은 20%만 표시
```

**핵심**: 페이지 이동하면 SSE 연결이 끊겨서 이벤트를 못 받음

---

## 해결

### 3가지 안전장치 추가

| 장치 | 설명 |
|------|------|
| 1. 연결 유지 | 페이지 이동해도 SSE 연결 안 끊음 |
| 2. 이벤트 저장 | 받을 사람 없으면 버퍼에 저장 (최대 1분) |
| 3. DB 조회 | 페이지 복귀 시 DB에서 최신 상태 확인 |

### 동작 방식

```
1. 문서 목록 페이지에서 업로드 시작 (20% 표시)
2. 다른 페이지로 이동 → SSE 연결 유지!
3. 백엔드가 40%, 60%, 80%, 100% 이벤트 보냄 → 버퍼에 저장
4. 문서 목록 페이지로 복귀
   → 버퍼에서 이벤트 받음
   → DB에서 최신 상태 확인
   → 100% 표시 ✅
```

---

## 왜 근본적 해결인가?

1. **이벤트 손실 방지**: 연결 끊지 않고 버퍼에 저장
2. **DB가 진실**: 이벤트 손실되어도 DB에서 복구
3. **모든 케이스 커버**: 페이지 이동, 탭 종료, 네트워크 오류 모두 대응

---

## 테스트 방법

1. 문서 업로드 시작
2. 업로드 진행 중 다른 페이지로 이동
3. 문서 목록으로 복귀
4. F5 없이 진행률 정상 표시되면 성공
