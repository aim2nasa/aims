### C-2. "AR", "CRS" 탭 — 약어 설명 전무 [Sora]

- **제안**: "마우스를 올리면 메뉴가 나타납니다"로 변경

### C-4. Tooltip `id="tooltip"` 하드코딩 — WCAG 위반 [Dana]
- **위치**: `shared/ui/Tooltip.tsx:229`
- **문제**: 모든 Tooltip이 동일한 `id="tooltip"` 사용. 한 페이지에 Tooltip 복수 존재 시 HTML ID 중복 → 접근성(WCAG) 위반, aria-describedby 오작동.
- **제안**: `useId()` 또는 인스턴스별 고유 ID 생성

### C-5. Input `id="input-error"` 하드코딩 [Dana]
- **위치**: `shared/ui/Input.tsx:98, 112`
- **문제**: 에러 메시지 요소가 `id="input-error"` 고정. 복수 Input 시 ID 충돌.
- **제안**: prop으로 id를 받거나 `useId()` 사용

## 3. MAJOR 이슈

### M-1. Header Progressive Disclosure — 핵심 기능 숨김 [Sora + Dana]
- **위치**: `HeaderView.tsx` 전체 구조

- **제안**: 검색 아이콘은 상시 표시, 또는 첫 방문 시 헤더 확장 상태로 시작

### M-2. "AND / OR" 프로그래밍 용어 노출 [Sora]
- **위치**: `DocumentSearchView.tsx:91-92`
- **문제**: 키워드 검색 모드에 "AND", "OR" 영어 약어 노출.
- **소라**: "AND가 뭐예요? 개발자들이 쓰는 말 아닌가요?"
- **제안**: "모두 포함" / "하나라도 포함"으로 변경

### M-3. "질문 검색" 크레딧 사용 사전 고지 부족 [Sora]
- **위치**: `DocumentSearchView.tsx:86, 1992`
- **문제**: 질문 검색이 크레딧(과금)을 사용하는데, 드롭다운에서는 알 수 없음.
- **소라**: "검색했더니 갑자기 크레딧이 차감됐어요! 미리 알려줬어야죠."
- **제안**: "질문 검색 (크레딧 사용)" 표시, 잔여 크레딧 노출

### M-4. "고객별 문서함" vs "고객문서분류함" — 유사 이름 [Sora] ✅ 스킵
- **위치**: `CustomMenu.tsx:575`, `docs/PAGE_NAMES.md:10`
- **문제**: 전체 고객 문서 탐색 vs 특정 고객 문서 분류인데 이름이 비슷.
- **소라**: "문서함이 두 개인데 뭐가 다른 거야?"
- **제안**: "고객 폴더 전체" vs "서류 분류함"으로 차별화
- **결론**: 스킵 — 두 기능은 완전히 다른 UI 맥락(좌측 메뉴 vs 고객상세페이지 내 버튼)에서 나타나므로 실제 혼동 가능성 없음. 이름 변경 시도가 오히려 억지스러움.

### M-6. CenterPaneView 닫기/뒤로가기 어포던스 부재 [Dana] ✅ 스킵
- **위치**: CenterPaneView 전체

### M-7. CustomMenu collapsed/expanded 데이터 이중 관리 [Dana]
