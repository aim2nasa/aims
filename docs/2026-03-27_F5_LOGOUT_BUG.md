# 이슈 보고서: F5 새로고침 시 로그아웃 버그

> 작성일: 2026-03-27 00:15 KST
> 상태: **미해결 — 재현 확인**
> 심각도: HIGH

---

## 현상

전체 문서 보기 페이지(`aims.giize.com/?view=documents-library`)에서 **F5 키로 새로고침하면 로그인 페이지로 이동**됨. 세션이 유지되지 않고 로그아웃 처리됨.

## 재현 절차

1. AIMS 로그인 (카카오 → PIN)
2. 전체 문서 보기 페이지 이동
3. F5 (브라우저 새로고침)
4. → `aims.giize.com/login` 로그인 페이지로 리다이렉트

## 영향

- 문서 처리 상태 확인 중 F5를 누르면 로그아웃 → 재로그인 필요
- 사용자 UX 심각 저하
- 문서 처리 완료 확인을 위한 새로고침이 불가능

## 원인 추정

- 세션/토큰 저장 방식 문제 (메모리 기반 vs localStorage/cookie)
- 배포 후 인증 토큰 무효화
- SPA 라우팅과 서버 라우팅 충돌

## 관련 파일 (추정)

- 프론트엔드 인증 관련 컴포넌트
- 카카오 OAuth 세션 관리
