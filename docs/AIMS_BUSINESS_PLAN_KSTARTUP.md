# AIMS 사업계획서 (K-Startup 예비창업자 프로그램)

## 1. 창업 아이템 개요

### 1.1 아이템명
**AIMS (Agent Intelligent Management System)**
- 보험 설계사를 위한 AI 기반 지능형 문서·고객 관리 플랫폼

### 1.2 사업 비전
> "보험 설계사의 행정 업무를 80% 줄이고, 고객 상담 시간을 2배로 늘린다"

### 1.3 핵심 가치 제안
| 기존 방식 | AIMS 솔루션 |
|-----------|-------------|
| 문서 수동 분류 (30분/건) | AI 자동 분류 (3초/건) |
| 계약 정보 수기 입력 | 연간보고서 자동 파싱 |
| 키워드 검색만 가능 | AI 의미 검색 (자연어 질의) |
| 고객 정보 엑셀 관리 | 통합 CRM + 관계도 시각화 |
| 설계사 개별 서버 필요 | 클라우드 SaaS 즉시 이용 |

## 2. 문제 정의 및 시장 분석

### 2.1 해결하고자 하는 문제

**보험 설계사의 일상적 고충:**

1. **문서 홍수**
   - 설계사 1인당 연간 2,000~5,000건 문서 처리
   - 청구서, 증권, 연간보고서, 진단서 등 다양한 형식
   - 수동 분류·검색에 전체 업무 시간의 40% 소요

2. **데이터 파편화**
   - 고객 정보: 엑셀 / 보험사 시스템 / 개인 메모 분산
   - "김철수 고객의 지난해 보험료?" → 5개 파일 뒤져야 확인

3. **연간보고서 처리의 비효율**
   - 보험사마다 다른 양식 (삼성생명, 한화생명, DB손해 등)
   - 1건 수기 입력에 15~30분 소요
   - 오류 발생 시 계약 관리 차질

4. **경쟁 도구의 한계**
   - 보험사 제공 시스템: 자사 상품만 지원
   - 범용 CRM: 보험 도메인 특화 기능 부재
   - 문서 관리 솔루션: AI 검색·분석 기능 미흡

### 2.2 시장 규모

#### 국내 시장 (TAM/SAM/SOM)

| 구분 | 규모 | 산출 근거 |
|------|------|----------|
| **TAM** (전체 시장) | 3,600억원 | 보험 IT 솔루션 시장 (2024) |
| **SAM** (유효 시장) | 720억원 | 설계사 업무 자동화 시장 (TAM의 20%) |
| **SOM** (목표 시장) | 36억원 | 초기 5년 목표 (SAM의 5%) |

#### 타겟 고객
- **1차 타겟**: 전속 설계사 (보험사 소속) - 약 18만명
- **2차 타겟**: GA(독립대리점) 설계사 - 약 12만명
- **3차 타겟**: 보험 GA 법인 (약 2,000개사)

#### 시장 트렌드
- InsurTech 시장 연평균 성장률: 12.5% (2023-2028)
- AI 문서 처리 시장: 연평균 35% 성장
- 클라우드 SaaS 전환 가속화

### 2.3 고객 페르소나

**페르소나 A: 김영희 (32세, 전속 설계사 5년차)**
- 월 40건 신규 계약, 관리 고객 500명
- 고충: "고객 문서 찾는데 하루 2시간 소요"
- 니즈: 빠른 검색, 자동 분류, 모바일 접근

**페르소나 B: 박민수 (45세, GA 소속 15년차)**
- 5개 보험사 상품 취급, 관리 고객 1,200명
- 고충: "연간보고서 30건 처리에 이틀 소요"
- 니즈: 멀티 보험사 지원, 일괄 처리, 자동 파싱

### 3.2 핵심 기능

#### 기능 1: AI 문서 자동 처리
```
문서 업로드 → 자동 분류 → OCR 텍스트 추출 → AI 요약·태깅 → 벡터 임베딩
     (3초)       (AI)         (Upstage)        (GPT-4)       (검색 준비)
```

**기술 스택:**
- OCR: Upstage Document AI (한글 인식률 99%+)
- 분류: GPT-4 기반 문서 유형 자동 판별
- 임베딩: OpenAI text-embedding-3-small (1536차원)

#### 기능 2: 연간보고서 자동 파싱
- 보험사별 양식 자동 인식 (삼성, 한화, DB, 현대 등)
- 계약 정보 구조화 추출 (증권번호, 보험료, 피보험자 등)
- 기존 고객·계약 데이터와 자동 매칭

**기술적 차별점:**
- Vision Language Model (GPT-4V) 활용
- 테이블 구조 자동 인식
- 오류 검증 로직 내장

#### 기능 3: 하이브리드 AI 검색
```
사용자 질문: "김철수 고객의 2024년 보험료 납입 내역은?"
                              ↓
┌─────────────────────────────────────────────────────────┐
│               하이브리드 검색 엔진                        │
│                                                         │
│  ┌────────────────┐     ┌────────────────┐            │
│  │ Entity Search  │     │ Vector Search  │            │
│  │ "김철수", "2024"│     │ "보험료 납입"   │            │
│  │  (MongoDB)     │     │  (Qdrant)      │            │
│  └───────┬────────┘     └───────┬────────┘            │
│          └──────────┬───────────┘                      │
│                     ↓                                   │
│           Cross-Encoder 재순위화                        │
│                     ↓                                   │
│              GPT-4 답변 생성                            │
└─────────────────────────────────────────────────────────┘
                              ↓
답변: "김철수 고객님의 2024년 보험료 납입 내역입니다:
      - 삼성생명 종신보험: 월 125,000원 (12회 납입)
      - 현대해상 실손보험: 월 45,000원 (12회 납입)
      총 납입액: 2,040,000원"
```

#### 기능 4: 고객 관계 관리 (CRM)
- 가족/직장 관계 시각화 (관계도 그래프)
- 지역별 고객 분포 (네이버 지도 연동)
- 계약 현황 대시보드
- 휴면 고객 관리
