# AIMS E2E 기능 테스트 시나리오

> **버전**: v1.0.0  
> **최종 수정**: 2026-04-02  
> **대상 환경**: https://localhost:5177 (개발 서버)  
> **도구**: Playwright MCP (--ignore-https-errors)  
> **계정**: aim2nasa@gmail.com / PIN 3007

---

## 테스트 레벨 정의

| 레벨 | 설명 | 검증 방식 |
|------|------|----------|
| **L1** | 페이지 접근/로드 확인 | URL 이동 + 스크린샷 |
| **L2** | UI 상호작용 | 클릭/입력 → 화면 변화 확인 |
| **L3** | 데이터 정합성 | 화면 숫자 vs DB 조회 비교 |
| **L4** | 엣지케이스/에러 처리 | 빈 값, 특수문자, 경계값 |

> 현재 v1.0.0은 **L1 수준**. 이후 버전에서 L2~L4를 점진 추가.

---

## 사전 조건

1. 프론트엔드 dev 서버 실행 중 (`cd frontend/aims-uix3 && npm run dev`)
2. Playwright MCP `--ignore-https-errors` 옵션 적용
3. 카카오 로그인 → PIN 입력 → 메인 화면 진입 완료
