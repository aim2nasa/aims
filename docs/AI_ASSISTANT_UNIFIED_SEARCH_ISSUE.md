# unified_search AI 검색 결과 부정확 이슈

> **발견 일시**: 2026-03-16
> **심각도**: Major
> **상태**: 미해결
> **발견 경로**: Playwright E2E 테스트 (Q-019, Q-020)

---

## 1. 증상

unified_search 도구가 키워드 검색과 AI(시맨틱) 검색 결과를 동시에 반환할 때, **AI 검색 결과가 쿼리와 무관한 문서를 반환**한다.

특히 **세무조정계산서가 어떤 쿼리에서든 AI 검색 상위에 고정적으로 출현**하는 현상이 관찰된다.

## 3. 원인 분석 (추정)

### 아키텍처
```
unified_search (MCP 도구)
├─ 키워드 검색: SmartSearch (:8100) → MongoDB 텍스트 매칭
└─ AI 검색: RAG API (:8000) → Qdrant 벡터 검색 + Cross-Encoder 재랭크
```

### 추정 원인

1. **임베딩 편향**: 세무조정계산서 문서의 벡터가 Qdrant에서 다양한 쿼리와 높은 코사인 유사도를 가짐
   - 이유: 세무조정계산서는 텍스트 길이가 길고(5000자+), 회사명/금액/날짜 등 범용적 정보가 밀집
   - text-embedding-3-small 모델이 긴 문서에 대해 범용적 벡터를 생성하는 경향

2. **Cross-Encoder 재랭크 실패**: reranker가 세무조정계산서에 높은 점수를 부여
   - 쿼리 "보험증권"과 세무조정계산서의 cross-encoder 점수가 0.99+ (rerank_score 기준)
   - 모든 쿼리에 대해 유사하게 높은 점수 → 재랭크 효과 없음

3. **사용자 필터만 적용**: Qdrant 검색 시 `owner_id` 필터만 있고, `document_type` 등 추가 필터 없음

### 검증 필요 항목
- [ ] Qdrant에서 "보험증권" 쿼리로 직접 검색 시 세무조정계산서 점수 확인
- [ ] Cross-Encoder rerank 전후 점수 비교
- [ ] 세무조정계산서의 청크 수 / 벡터 밀도 확인

---

## 4. 영향 범위

- unified_search의 AI 검색 결과를 표시하는 모든 질문에 영향
- 키워드 검색이 정확하면 사용자가 원하는 문서를 찾을 수 있으므로 심각도는 Major (Critical 아님)
- 다만 AI 검색 결과가 항상 동일 문서를 반환하면 사용자 신뢰도 저하

---

## 5. 해결 방안 (검토 필요)

| 방안 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A. 쿼리-문서 유사도 임계값 | AI 검색 결과 중 유사도 점수가 임계값 미만이면 제외 | 간단 | 임계값 튜닝 필요 |
| B. 키워드 검색 결과와 중복 제거 | AI 결과에서 키워드 결과와 겹치지 않는 것만 표시 | 다양성 확보 | 관련 문서도 제외될 수 있음 |
| C. document_type 필터 | 쿼리에서 문서 유형을 추론하여 Qdrant 필터에 추가 | 정확도 향상 | 쿼리 분석 로직 추가 필요 |
| D. 임베딩 품질 개선 | text-embedding-3-large로 업그레이드 또는 청크 크기 조정 | 근본 해결 | 비용 증가, 재임베딩 필요 |
