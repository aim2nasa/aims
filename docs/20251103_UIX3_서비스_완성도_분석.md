**현재 상황**: 사용자 사이트만 존재, 운영자 입장에서 문제를 파악하거나 관리할 수단 없음

| 항목 | 필요 도구 | 설명 |
|------|----------|------|
| **에러 추적** | Sentry | 사용자가 겪는 에러를 실시간으로 캐치하고 알림 |
| **사용량 모니터링** | 커스텀 tracker | AI 토큰, 스토리지, API 호출 사용량 추적 및 비용 계산 |
| **과금 시스템** | Stripe/Toss | 구독 플랜 관리, 결제 수단 등록, 청구서 자동 생성 |
| **운영자 대시보드** | Admin 사이트 | 전체 사용자 관리, 서버 상태, 트래픽 모니터링, KPI |
| **서비스 상태 페이지** | Statuspage.io | 서비스 정상/장애 표시, 장애 이력, 예정 점검 공지 |
| **데이터 백업** | MongoDB 백업 | 일별/주별 자동 백업, 장애 시 복구 (RTO 1시간, RPO 15분) |

| **부하 테스트** | k6, Artillery | 동시 접속자 수 테스트 (목표: 1000+ CCU) |

| **CI/CD** | GitHub Actions | Git push 시 자동 테스트/배포, 롤백 기능 |

## 🎯 즉시 실행 권장 (1주 내, 무료)

### Tier 1: 필수 (출시 전 반드시 구현)

1. **Sentry 연동** - 1일
   - 무료 플랜으로 시작 가능
   - 프론트엔드 에러 실시간 추적
   - Slack 알림 연동

2. **MongoDB 백업** - 1일
   - cron + mongodump 스크립트
   - S3 또는 Backblaze B2에 저장
   - 복원 테스트 수행

3. **환경 변수 관리** - 1일
   - .env.example, .env.development, .env.production 생성
   - 하드코딩된 API URL 제거
   - vite.config.ts 수정

4. **전역 에러 바운더리** - 1일
   - GlobalErrorBoundary 컴포넌트 생성
   - main.tsx에 적용
   - Sentry 연동

5. **CI/CD 파이프라인** - 3일
   - GitHub Actions workflow 작성
   - 자동 테스트 (npm test)
   - 자동 배포 (Docker build & deploy)
   - 배포 실패 시 Slack 알림

**총 소요 시간**: 1주
**총 비용**: $0/월

**효과**: 프로덕션 배포 가능 수준 도달 (현재 불가 → 가능)
