# 고객/계약 일괄등록 엑셀 파일 규격 (v0.1)

> **문서 버전**: v0.1 | **최종 수정**: 2025-12-04
>
> 이 문서는 진화적으로 수정됩니다. 변경 이력은 하단 버전 정보를 참조하세요.

## 개요

AIMS 시스템에서 고객과 계약 정보를 일괄 등록하기 위한 엑셀 파일(.xlsx) 표준 규격입니다.

---

## 파일 구조

엑셀 파일은 최대 3개의 시트로 구성됩니다:

| 시트명 | 용도 | 필수 여부 |
|--------|------|----------|
| 개인고객 | 개인 고객 정보 등록 | 선택 |
| 법인고객 | 법인 고객 정보 등록 | 선택 |
| 계약 | 계약 정보 등록 | 선택 |

> **참고**: 최소 1개 이상의 시트가 필요합니다.

---

## 시트별 컬럼 규격

### 1. 개인고객 시트

#### 필수 컬럼

| 컬럼명 | 인식 패턴 | 설명 |
|--------|----------|------|
| 고객명 | `고객명`, `이름`, `성명`, `고객` | 고객 이름 (빈값 불가) |

#### 선택 컬럼

| 컬럼명 | 설명 | 형식 |
|--------|------|------|
| 연락처 | 휴대폰 번호 | `010-1234-5678` |
| 주소 | 거주지 주소 | 자유 형식 |
| 성별 | 성별 | `남` / `여` |
| 생년월일 | 생년월일 | `YYYY-MM-DD` |
| 이메일 | 이메일 주소 | `example@email.com` |

---

### 2. 법인고객 시트

#### 필수 컬럼

| 컬럼명 | 인식 패턴 | 설명 |
|--------|----------|------|
| 고객명 | `고객명`, `이름`, `성명`, `고객` | 법인명 (빈값 불가) |

#### 선택 컬럼

| 컬럼명 | 설명 | 형식 |
|--------|------|------|
| 연락처 | 대표 전화번호 | `02-1234-5678` |
| 주소 | 사업장 주소 | 자유 형식 |
| 사업자번호 | 사업자등록번호 | `123-45-67890` |
| 대표자명 | 대표자 이름 | 자유 형식 |

---

### 3. 계약 시트

#### 필수 컬럼

| 컬럼명 | 인식 패턴 | 설명 | 검증 내용 |
|--------|----------|------|----------|
| 고객명 | `고객명`, `이름`, `성명`, `고객` | 계약자 이름 | 빈값 체크 |
| 상품명 | `상품명`, `상품`, `보험명` | 보험 상품명 | DB 매칭 검증 |
| 계약일 | `계약일` | 계약 체결일 | 날짜 형식 검증 |
| 증권번호 | `증권번호`, `policy` | 보험 증권번호 | 빈값 + 중복 체크 |

#### 선택 컬럼

| 컬럼명 | 설명 | 형식 |
|--------|------|------|
| 보험료 | 납입 보험료 | 숫자 |
| 납입주기 | 보험료 납입 주기 | `월납`, `연납`, `일시납` 등 |
| 이체일 | 보험료 이체일 | `1` ~ `31` |
| 만기일 | 계약 만기일 | `YYYY-MM-DD` |
| 피보험자 | 피보험자 이름 | 자유 형식 |
| 수익자 | 수익자 이름 | 자유 형식 |

## 컬럼 인식 규칙

컬럼명에 아래 키워드가 **포함**되면 자동으로 해당 타입으로 인식됩니다 (대소문자 무시).

| 검증 타입 | 인식 키워드 |
|----------|------------|
| 고객명 (customerName) | `고객명`, `이름`, `성명`, `고객` |
| 상품명 (productName) | `상품명`, `상품`, `보험명` |
| 계약일 (contractDate) | `계약일` |
| 증권번호 (policyNumber) | `증권번호`, `policy` |

### 예시

- `고객명` → customerName 인식
- `계약자 이름` → customerName 인식 (`이름` 포함)
- `보험상품명` → productName 인식 (`상품명` 포함)
- `계약일자` → contractDate 인식 (`계약일` 포함)

## 검증 프로세스

### 검증 순서

```
1. 개인고객 검증 → 2. 법인고객 검증 → 3. 계약 검증 → 4. 일괄등록
```

### 시트별 검증 내용

| 시트 | 검증 항목 |
|------|----------|
| 개인고객 | 고객명 빈값 체크 |
| 법인고객 | 고객명 빈값 체크, 개인/법인 간 동명이인 검증 |
| 계약 | 고객명·상품명·계약일·증권번호 검증 |

### 상품명 매칭 상태

| 상태 | 설명 | 표시 |
|------|------|------|
| 원본 매칭 | DB에 정확히 일치하는 상품명 | 녹색 |
| 수정 매칭 | 공백/대소문자 정규화 후 매칭 | 노란색 |
| 미매칭 | DB에 없는 상품명 | 빨간색 |

## 날짜 형식

### 지원 형식

시스템에서 자동 변환되는 날짜 형식:

| 입력 형식 | 변환 결과 |
|----------|----------|
| `2024-03-15` | `2024-03-15` (그대로) |
| `3/15/24` | `2024-03-15` |
| `3/15/2024` | `2024-03-15` |
| `2024/03/15` | `2024-03-15` |

### 권장 형식

```
YYYY-MM-DD (예: 2024-03-15)
```

## 주의사항

1. **첫 번째 행은 컬럼 헤더**로 인식됩니다.
2. **시트명**은 정확히 `개인고객`, `법인고객`, `계약`으로 지정해야 합니다.
3. **동명이인 주의**: 개인고객과 법인고객 시트에 동일한 이름이 있으면 검증 단계에서 경고가 표시됩니다.
4. **증권번호는 고유**해야 합니다. 중복된 증권번호는 오류로 처리됩니다.
5. **상품명**은 시스템에 등록된 보험상품과 일치해야 합니다.
