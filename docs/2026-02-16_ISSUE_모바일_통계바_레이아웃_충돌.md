# Container Query vs Media Query 충돌로 인한 모바일 레이아웃 깨짐

**날짜**: 2026-02-16
**파일**: `frontend/aims-uix3/src/components/CustomerViews/CustomerRegionalView/RegionalTreeView.css`
**커밋**: `9b283835`

---

## 증상

지역별 고객 뷰(RegionalTreeView) 모바일 모드(360x740)에서 통계바가 깨짐:
- `[744명]` 버튼이 전체 폭을 차지
- `[전체 지역 ▼]`과 `[전체 구/군 ▼]` 드롭다운이 찌그러져 텍스트 잘림
- 의도한 1줄 레이아웃: `[744명] [전체 지역 ▼] [전체 구/군 ▼]`

## 근본 원인: Container Query와 Media Query의 충돌

### 충돌 구조

```
┌─ @container (max-width: 900px) ─────────────────────────┐
│  .stat-item:first-child { flex-basis: 100%; }           │
│  → PC에서 RightPane 열릴 때 2줄 레이아웃 전환용          │
│  → 컨테이너 폭 < 900px일 때 적용                        │
└─────────────────────────────────────────────────────────┘

┌─ @media (max-width: 480px) ─────────────────────────────┐
│  .regional-tree-stats { flex-wrap: nowrap; }            │
│  → 모바일에서 1줄 컴팩트 레이아웃 강제                   │
│  → 뷰포트 폭 ≤ 480px일 때 적용                         │
└─────────────────────────────────────────────────────────┘
```

**핵심**: 모바일(360px)에서는 **두 규칙이 동시에 적용**됨.
- 컨테이너 폭(360px) < 900px → container query 적용 → `flex-basis: 100%`
- 뷰포트 폭(360px) < 480px → media query 적용 → `flex-wrap: nowrap`

결과: `flex-wrap: nowrap`으로 줄바꿈은 막았지만, 첫 번째 아이템이 `flex-basis: 100%`를 가지므로 나머지 아이템의 공간이 없음. flex shrink로 겨우 표시되지만 드롭다운 텍스트가 잘림.

### 왜 발견이 어려웠나

1. **Container Query는 미디어 쿼리와 다른 위치에 정의됨** - 모바일 `@media` 블록만 보면 원인을 알 수 없음
2. **Container Query는 뷰포트가 아닌 컨테이너 크기 기준** - 모바일에서도 항상 트리거됨
3. **flex-basis와 flex-wrap의 상호작용** - `nowrap` 상태에서 `flex-basis: 100%`가 어떤 효과를 내는지 직관적이지 않음

## 수정 내용

모바일 `@media (max-width: 480px)` 블록에 container query 오버라이드 추가:

```css
@media (max-width: 480px) {
  /* @container(max-width:900px)의 flex-basis:100% 오버라이드 */
  .stat-item:first-child {
    flex-basis: auto;
  }
}
```

## 교훈

### Container Query 사용 시 주의사항

1. **Container Query는 모든 크기에서 평가됨** - `@container (max-width: 900px)`은 PC에서만 적용되는 것이 아니라, 컨테이너가 900px 미만인 모든 환경(모바일 포함)에서 적용됨

2. **Media Query와 Container Query는 독립적** - 서로 다른 축(뷰포트 vs 컨테이너)을 기준으로 하므로, 하나를 수정해도 다른 하나가 간섭할 수 있음

3. **모바일 레이아웃 디버깅 시 Container Query도 확인 필수** - `@media` 블록만 보면 안 됨. 같은 셀렉터에 영향을 주는 `@container` 규칙이 있는지 반드시 검색할 것

### 검색 패턴

```bash
# 특정 셀렉터에 영향을 주는 모든 규칙 찾기
grep -n "stat-item:first-child" RegionalTreeView.css
grep -n "@container" RegionalTreeView.css
```
