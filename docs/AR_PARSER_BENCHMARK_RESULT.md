## 1. 테스트 목적

AR(Annual Report) 문서 파싱에 사용할 최적의 방법을 **객관적 데이터**로 선정하기 위함.

현재 시스템은 **OpenAI API**를 사용 중이나, 다음 의문이 제기됨:
- 정말 OpenAI가 최선인가?
- 더 빠르고 저렴한 대안은 없는가?
- 정확도는 어떻게 비교되는가?

---

## 2. 비교 대상 (4가지 방법)

### 2.1 OpenAI (현재 사용 중)
- **모델**: gpt-4.1
- **방식**: PDF를 OpenAI에 업로드 → Chat Completions API로 JSON 추출
- **비용**: ~$0.05-0.15/건 (토큰 비용)
- **특징**: LLM 기반, 비정형 데이터도 처리 가능

### 2.2 pdfplumber
- **버전**: 0.10.x
- **방식**: Python 라이브러리로 PDF 텍스트/표 직접 추출
- **비용**: 무료 (오픈소스)
- **특징**: 로컬 실행, 매우 빠름

### 2.3 Camelot
- **버전**: 1.0.x
- **방식**: 표 추출 전문 라이브러리 (lattice/stream 모드)
- **비용**: 무료 (오픈소스)
- **특징**: 표 구조 인식에 특화

### 2.4 Upstage Document AI
- **API**: Document Parse API
- **방식**: 클라우드 OCR + 문서 구조 분석
- **비용**: ~$0.01/페이지
- **특징**: 한국어 문서에 최적화
