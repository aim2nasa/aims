# 업로드 완료 후 행동 유도(CTA) UX 개선 기획안

**작성일**: 2026-03-26
**상태**: 기획 확정 (Dana 리뷰 반영 v2)

---

## 문제 정의

업로드 완료 후 사용자가 다음에 무엇을 해야 하는지 알기 어렵다.
"확인" 버튼은 모달만 닫을 뿐, 다음 행동을 안내하지 않는다.

## 현재 상태

| 페이지 | 완료 후 버튼 | 문제점 |
|--------|-------------|--------|
| 문서 등록 | "새 문서 등록", "전체 문서 보기" | **이미 해결됨** — 변경 불필요 |
| 문서 일괄등록 | "처리 상태 보기", "확인" | "확인"은 모달 닫기만 수행. 다음 행동 불명확 |
| 고객 일괄등록 | "전체 고객/계약 보기" 링크 | 로그 텍스트 안에 묻혀 발견 어려움 |

## 개선 대상

1. ~~문서 등록~~ — 이미 구현 완료, 변경 불필요
2. **문서 일괄등록** — BatchDocumentUploadView > UploadSummary
3. **고객 일괄등록** — ContractImportView > ExcelRefiner

## 개선안

### 와이어프레임

```
┌──────────────────────────────────────────────┐
│  ✓ 업로드 완료                                │
│  8개 파일이 3명의 고객에게 등록되었습니다.         │
│                                              │
│  (기존 통계/결과 영역 유지)                      │
│                                              │
│  [계속 일괄등록]              [전체 문서 보기]    │
│   (Secondary)                  (Primary)      │
└──────────────────────────────────────────────┘
```

- 안내 문구("다음 작업을 선택하세요" 등) 없음 — 버튼 라벨만으로 충분
- Primary 버튼은 오른쪽 배치 (Apple HIG 준수)

### 시나리오별 CTA — 문서 일괄등록

| 시나리오 | 왼쪽 (Secondary) | 오른쪽 (Primary) |
|----------|-----------------|-----------------|
| 전체 성공 | 계속 일괄등록 | 전체 문서 보기 |
| 부분 실패 (재시도 가능) | 전체 문서 보기 | 실패 항목 재시도 |
| 부분 실패 (재시도 불가) | 계속 일괄등록 | 전체 문서 보기 |
| 전체 실패 | 돌아가기 | 다시 시도 |
| 취소됨 | 전체 문서 보기 | 계속 일괄등록 |

### 시나리오별 CTA — 고객 일괄등록

| 시나리오 | 왼쪽 (Secondary) | 오른쪽 (Primary) | 비고 |
|----------|-----------------|-----------------|------|
| 성공 (고객+계약) | 계속 일괄등록 | 전체 고객 보기 | 안내 텍스트에 "전체 계약 보기" 링크 포함 |
| 성공 (고객만) | 계속 일괄등록 | 전체 고객 보기 | |
| 부분 실패 | 계속 일괄등록 | 전체 고객 보기 | 오류 목록은 기존 결과 영역에 표시 |
| 전체 실패 | 돌아가기 | 다시 시도 | |

## 설계 원칙

- 기존 "확인" 버튼 제거 — 의미 없는 닫기 대신 명확한 행동 유도
- Primary 버튼은 오른쪽 (Apple HIG 준수)
- 버튼 라벨만으로 행동 전달 — 별도 안내 문구 불필요
- 기존 통계/결과 영역은 그대로 유지 (하단에 CTA 버튼 배치)
- 고객 일괄등록의 "계속 일괄등록"은 sessionStorage 포함 전체 상태 초기화
