## 높은 임팩트 — 기존 인프라로 빠르게 구현 가능

### 1. 고객 리마인더/일정 알림

설계사에게 가장 중요한 것은 **타이밍**이다.

- 계약 만기 30/60/90일 전 자동 알림
- 고객 생일, 경조사 리마인더
- AR/CRS 발행 시점 자동 알림 ("이 고객의 새 AR이 나왔습니다")
- 이미 계약 데이터와 고객 데이터가 있으므로 **백엔드 크론 + 프론트 알림 UI**만 추가하면 구현 가능

### 2. 고객 미팅 노트

미팅 전후를 지원하는 기능:

- 미팅 전: AI가 고객 요약 브리핑 자동 생성 (계약 현황, 최근 변동, 메모 이력)
- 미팅 후: 음성 녹음 → 텍스트 변환 → 요약 → 고객 메모에 자동 저장
- Whisper API 또는 DGX Spark 로컬 추론으로 구현 가능

---

## 중간 임팩트 — 차별화 강화

### 3. 고객 포트폴리오 대시보드

설계사의 전체 포트폴리오를 경영자 시점으로:

- 월별 보험료 수입 추이 (신규/유지/해약)
- 고객 세그먼트별 분포 (연령대, 보험료 규모, 계약 수)
- "이번 달 만기 도래 고객 15명" — 액션 아이템 자동 생성
- 현재 통계 대시보드를 확장하는 형태로 구현 가능

### 4. 스마트 문서 비교

같은 고객의 AR을 연도별로 자동 비교:

- "작년 대비 보험료가 12% 올랐습니다"
- "2024년에 있던 A계약이 2025년에 사라졌습니다 (해약 추정)"
- 이미 시계열 추적이 언급되어 있는데, 이것을 **시각적 diff 뷰**로 발전

### 5. 엑셀/PDF 리포트 내보내기

설계사가 고객에게 직접 보여줄 수 있는 자료 생성:

- 고객별 보장 현황 요약 PDF
- 가족 단위 보험 현황 리포트
- 설계사 브랜딩(로고/이름)이 들어간 깔끔한 양식
- "AIMS로 만든 리포트를 고객에게 보여주는 순간"이 바이럴 포인트

---

## 장기적 — 경쟁 해자 구축

### 6. 고객 이벤트 타임라인

고객과의 모든 상호작용을 시간순으로:

- 문서 업로드, 계약 변동, 메모, 미팅 기록, AI 질의 이력
- "이 고객과 언제 무슨 이야기를 했는지" 한눈에 파악
- CRM의 핵심이지만, **문서 중심 타임라인**이라는 점에서 차별화

### 7. 설계사 간 익명 벤치마크

개인정보 없이 통계 수준의 비교:

- "당신의 고객 1인당 평균 계약 수는 2.3건, 상위 20% 설계사는 4.1건"
- "당신의 고객 평균 보험료는 월 15만원, 동일 경력 설계사 평균은 22만원"
- 자극과 동기부여 → **앱 재방문율 증가**

### 8. 카카오톡/문자 연동

설계사의 주요 소통 채널인 카카오톡과 연결:

- AIMS에서 바로 고객에게 메시지 발송
- 생일/만기 축하 메시지 템플릿
- 카카오 비즈메시지 API 활용

## 전체 기능 비교표

| # | 기능 | 카테고리 | 임팩트 | 구현 난이도 | 예상 기간 | 활용 데이터 | 핵심 가치 |
|---|------|----------|--------|------------|-----------|------------|-----------|
| 1 | 고객 리마인더/일정 알림 | 고객 관리 | 높음 | 낮음 | 2~3주 | 계약 만기일, 고객 생일 | 리텐션 (일상적 사용 유도) |
| 2 | 고객 미팅 노트 | AI 어시스턴트 | 높음 | 중간 | 3~4주 | 계약·메모·문서 이력 | 생산성 (미팅 준비 시간 절감) |
| 3 | 고객 포트폴리오 대시보드 | 데이터 시각화 | 중간 | 중간 | 4~5주 | 전체 고객·계약 통계 | 경영 인사이트 (포트폴리오 조감) |
| 4 | 스마트 문서 비교 | AI 분석 | 중간 | 중간 | 3~4주 | 연도별 AR 파싱 데이터 | 인사이트 (변동 자동 감지) |
| 5 | 엑셀/PDF 리포트 내보내기 | 문서 생성 | 중간 | 낮음 | 2~3주 | 고객·계약·보장 현황 | 바이럴 (고객 대면 시 활용) |
| 6 | 고객 이벤트 타임라인 | 고객 관리 | 중간 | 중간 | 4~5주 | 문서·계약·메모·AI 이력 | 관계 관리 (상호작용 전체 조회) |
| 7 | 설계사 간 익명 벤치마크 | 데이터 시각화 | 낮음 | 높음 | 6~8주 | 다수 설계사 통계 (익명) | 동기부여 (재방문율 증가) |
| 8 | 카카오톡/문자 연동 | 커뮤니케이션 | 중간 | 높음 | 6~8주 | 고객 연락처, 계약 만기 | 편의성 (소통 채널 통합) |

> **제외**: 고객 보장 갭 분석 — 보험사가 자사 계약 데이터로 더 정확하게 수행할 수 있는 영역. AIMS는 보험사가 할 수 없는 "고객 중심 정보 관리"에 집중.

> **논의 필요 (2026-02-27)**: 고객 리마인더/알림 — 계약 기반 알림(만기, 납입일)은 보험사 영역이므로 제외 대상. 단, 보험사가 모르는 정보(생일, 경조사, 상담 팔로업, 가족 상황 변화 등) 기반의 "관계 관리 리마인더"는 AIMS 고유 영역. 리포지셔닝 후 유지할지, 범위를 재정의할지 추가 논의 필요. **아이디어를 더 모아서 숙성시킨 뒤 결정.**

| **5** | 고객 포트폴리오 대시보드 | 기존 통계 대시보드 확장, 경영 인사이트 |
