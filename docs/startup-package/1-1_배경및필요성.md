# 1-1. 창업아이템 배경 및 필요성

> **버전 v0.2 (2026-02-25)** — 브레인스토밍 논의 반영 전면 개편

---

## 보험은 데이터 산업인데, 설계사는 데이터 빈곤층이다

보험사 본사에는 빅데이터 분석, AI 언더라이팅, 자동 심사 시스템이 있습니다. 수천억 원을 투입한 최첨단 IT 인프라가 갖춰져 있습니다.

같은 산업의 최전선에서 고객을 대면하는 설계사에게는? **종이 파일과 엑셀**이 있습니다.

같은 산업 내에서 본사와 현장의 디지털 격차가 이렇게 극단적인 산업이 또 있을까요? 이 역설의 뿌리를 파고 들어가면, 하나의 구조적 원인에 닿습니다.

---

## "계약"을 보는 보험사, "고객"을 보는 설계사

보험사와 설계사는 같은 산업에 있지만, **전혀 다른 것을 봅니다.**

보험사 시스템의 관심사는 **계약(Policy)**입니다. "증권번호 12345678의 보험료는 얼마인가", "이 계약의 위험률은 적정한가" — 보험사에게 중요한 단위는 **계약 한 건**입니다.

설계사의 관심사는 **고객(Customer)**입니다. "김철수라는 사람은 보장이 충분한가", "이 가족의 전체 보험 상황은 어떤가", "지난번 상담에서 뭘 약속했는가" — 설계사에게 중요한 단위는 **사람 한 명**입니다.

보험사는 수천억 원을 들여 IT 시스템을 구축하지만, 그 핵심 축은 처음부터 끝까지 **계약**입니다. 정작 현장에서 고객을 대면하는 설계사에게 필요한 것은 **고객 중심의 정보**인데, 주어진 것은 계약 조회 화면 하나뿐입니다.

**이 간극은 추상적인 것이 아닙니다.** 두 가지 구체적인 사례로 보여드리겠습니다.

### 사례 1: 공간의 간극 — "가족 전체를 한눈에 볼 수 없다"

설계사에게 자연스러운 관리 단위는 **가족**과 **법인**입니다. "김철수 가족 전체 보험 현황", "ABC법인 임직원 단체보험" — 이것이 설계사가 고객을 바라보는 방식입니다.

그러나 보험사 시스템은 **계약 1건 단위 조회만 가능**합니다. 가족 구성원 각각의 계약을 따로따로 조회하고, 결과를 수동으로 엑셀에 정리해야 합니다. 법인도 마찬가지 — 직원 수만큼 반복 조회하고 수작업으로 통합해야 합니다.

- 보험사: "증권번호 12345678" → 조회 가능
- 설계사: "김철수네 가족 전체" → **조회 불가능**

### 사례 2: 시간의 간극 — "어제를 알 수 없다"

설계사는 고객과 **길게는 20년 이상, 평생**의 관계를 가져갑니다. 이 긴 시간 동안의 계약 이력과 변화를 파악하는 것이 상담의 기본입니다.

그런데 보험사 시스템은 **조회 시점의 현재 계약만 보여줍니다.** 모든 보험사가 동일합니다. 해약된 계약, 취소된 계약은 조회할 수 없습니다.

극단적인 예를 들면, 고객이 과거에 해약한 **동일한 상품을 다시 추천하는 상황**이 발생합니다. "이 보험 전에 들었다가 해약했잖아요" — 고객의 신뢰는 순식간에 무너집니다. 설계사 잘못이 아닙니다. 보험사 시스템에 그 정보가 없으니 알 수가 없는 것입니다.

**보험사 시스템은 공간적으로도(가족·법인 묶기 불가), 시간적으로도(이력 추적 불가) 설계사의 관점을 지원하지 않습니다.**

## 설계사는 어떻게 버티고 있는가

체계적 도구가 없으니, 설계사들은 각자의 방식으로 관리합니다.

- **비서 고용** — 문제가 심각하니 아예 사람을 고용합니다. 그러나 비서도 같은 수작업을 대신할 뿐. 입력 오류, 누락, 버전 불일치가 발생하고, 비서가 바뀌면 관리 체계도 처음부터 다시. **인건비를 투입해도 근본 문제는 풀리지 않습니다.**
- **종이 파일** — 고객별 클리어 파일에 PDF 출력물 보관. 고객 100명이면 서랍장 통째로. 검색 불가.
- **개인 클라우드** — 네이버/구글 드라이브에 PDF 저장. 그러나 PDF **안의 내용**은 검색 안 됨.
- **엑셀** — 수동 입력. PDF와 미연결. 덮어쓰기 한 번이면 끝.
- **카카오톡 메모** — 나에게 보내기. 3개월 후 찾기 불가능.

어떤 방식이든 공통: 정보가 5곳 이상에 파편화, PDF 안의 데이터는 눈으로 읽는 수밖에 없고, 한 명의 고객에 대한 전체 그림을 한눈에 볼 수 없습니다. **문제는 사람이 부족한 것이 아니라, 도구가 없는 것입니다.**

## 생산성 손실의 실체

설계사의 시간은 곧 매출입니다.

| 손실 유형 | 규모 |
|---|---|
| 매월 고객별 PDF 수동 다운로드 (100명 기준) | **월 4~8시간** |
| 미팅 전 고객 정보 수집·정리 | 1건당 **20~30분** |
| 연간 문서 수집·정리 시간 | **100시간 이상** |
| 보험 관련 불완전판매 비율 (금감원 2024년) | 금융 제재 47건 중 **36건(76.6%)** |
| 설계사 13개월 정착률 (생명보험협회 2023년) | **47.3%** — 이직 시 데이터 전량 유실 |

연 100시간을 절약하면 고객 미팅 200건 추가 가능(1건 30분). 계약 전환율 10%만 잡아도 20건 신규 계약. **"편리하다"가 아니라 "돈이 된다"**는 것이 설계사에게 AIMS의 가치입니다.

## 왜 아무도 이 문제를 풀지 못했는가 — 4개의 벽

**1벽. 고객밀착형 데이터는 보험사의 영역이 아니다**

"둘째 아이 출산 예정이라 보장을 늘리고 싶다", "부친이 암 진단을 받아 본인도 걱정이 많다" — 이것은 고객이 설계사를 **믿고** 공유한 보안 데이터입니다. 보험사가 이를 수집하고 관리한다면 어느 고객도 원치 않을 것입니다. 다루기 힘든 것이 아니라, **다뤄서는 안 되는 영역**입니다.

게다가 보험사는 매우 보수적이어서, 설계사들의 니즈나 요구사항에 거의 대응하지 않습니다. IT 시스템 변경에 수개월~수년의 의사결정이 필요하고, 39만 설계사 각자의 현장 니즈를 본사가 일일이 대응하는 것은 구조적으로 불가능합니다.

**2벽. 범용 CRM은 보험 현장과 맞지 않는다**

Salesforce, HubSpot은 "보험 계약", "Annual Report", "펀드 수익률"이라는 도메인을 모릅니다. PDF 자동 파싱 기능도 없습니다. 커스터마이징에 IT 전문가가 필요하고, 개인 설계사가 월 수십만 원을 감당하기 어렵습니다.

**3벽. IT·AI 기술은 소규모 현장 직종에 닿지 않는다**

기술 혜택은 대기업 → 중견기업 → 스타트업 순으로 내려옵니다. 개인 단위로 영업하는 설계사에게까지 도달하지 않습니다. **기술이 존재하는 것과, 그 기술이 현장에 닿는 것은 완전히 다른 문제입니다.**

**4벽. 기술이 안 됐다 — 3년 전까지는**

PDF에서 데이터를 정확하게 뽑아내고, 내용을 이해하고, 자연어 질문에 답하는 것은 3년 전까지 정확도와 비용 양면에서 실용이 불가능했습니다.

## 왜 지금 — AI 기술의 성숙

| 기술 | 3년 전 (2022) | 현재 (2025) | 변화 |
|---|---|---|---|
| OCR (문서 텍스트 추출) | 정확도 85~90% | 99%+ | 실무 적용 가능 |
| LLM (문서 이해·요약) | 연구 단계 | GPT-4o 상용화 | 문서를 읽고 핵심 요약 |
| RAG (질의응답) | 개념 증명 수준 | 프로덕션 검증 | "보험료 얼마야?" → 즉답 |
| API 비용 | $0.06/1K토큰 | $0.00015/1K토큰 | **400배 하락** |
| 벡터 검색 | 대기업만 구축 가능 | 오픈소스 | 의미 기반 문서 검색 |

3년 전에는 수억 원이 들었던 것이, 지금은 월 수만 원으로 가능합니다. 그리고 이 변화는 서비스를 **이용하는 측**만이 아니라 **만드는 측**에도 일어났습니다. AI 기반 개발 도구의 등장으로, 1인 기업도 대규모 시스템을 구축할 수 있는 시대가 되었습니다.

## AIMS: "음영지역"에 다리를 놓다

> **AIMS는 IT·AI 기술의 음영지역에 있는 개인사업자 수준의 소규모 현장 직종에 다리를 놓아, 이들이 높은 생산성을 이루는 도구가 되고자 합니다.**

핵심 원리는 단순합니다:

> **"설계사가 문서를 올리면, AI가 읽고, 정리하고, 찾아준다."**

| 설계사의 고통 | AIMS가 해결하는 방법 |
|---|---|
| 매달 수백 건 PDF 수동 다운로드 | **자동 수집 엔진**: 전 고객 PDF 자동 다운로드 |
| PDF 열어서 숫자 하나하나 확인 | **AI 문서 파싱**: 계약정보 자동 추출, 테이블로 표시 |
| 고객 정보가 5곳에 흩어져 있음 | **통합 고객 관리**: 연락처·문서·계약·메모를 한 화면에 |
| "이 고객 보험료가 뭐였지?" | **AI 어시스턴트**: 자연어로 질문하면 3초 내 답변 |
| 가족/법인 단위로 묶어 볼 수 없음 | **관계 기반 조회**: 가족·법인 한 화면에서 통합 조회 |
| 과거 해약 이력을 알 수 없음 | **시계열 추적**: 연도별 보험 현황 변화 자동 누적·비교 |
| 이직하면 데이터 전부 유실 | **설계사 소유 플랫폼**: 본인이 데이터 소유 |

AIMS는 단순한 파일 정리 도구가 아닙니다. 고객 100명의 이력을 기억하고 찾아주는 **설계사의 두 번째 뇌**입니다. AIMS가 있으면 설계사는 **문서 정리자**에서 **고객 전문가**로 돌아갑니다.

기술 성숙, 시장 구조 전환, 규제 순풍 — 세 가지 흐름이 동시에 만나는 지점에 AIMS가 있습니다.
