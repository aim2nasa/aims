# AC PROD 진단 스크린샷 + 1단계 검증

**일시**: 2026-04-06
**프로세스**: Compact Fix

## 이슈

1. PROD에서 1단계 네비게이션이 실제로 어떤 화면에서 진행되는지 추적 불가
2. 이전 실행의 잔여 화면 상태로 1단계가 잘못 완료되어 2단계에서 크래시

## 원인

- PROD에서 정상 진행 중 스크린샷을 남기지 않음 (크래시 시에만 캡처)
- 1단계 완료 검증 없음 (click + sleep 후 무조건 "완료" 처리)

## 수정 내용

1. PROD 진단 스크린샷: CAPTURE_DIR/.diag/에 단계별 캡처
   - 정상 완료 시 자동 삭제 (흔적 없음)
   - 크래시 시 보존 (분석용)
   - 다음 실행 시 이전 세션 정리
2. 1단계 완료 검증: 초성 버튼(ㄱ) 존재 확인 → 미발견 시 1회 재시도

## 추가: 이전 단계 복귀 복구 전략 (2026-04-06)

### 문제
이미지를 찾지 못했을 때 같은 자리에서 대기만 하는 것은, 화면 상태 자체가 잘못된 경우 무의미.

### 해결
1단계 네비게이션을 함수로 추출하여, 실패 시 MetLife 로고(안전한 상태)로 복귀 후 전체 재시도.
2단계 초성 버튼 실패 시에도 1단계부터 복구.

- 래퍼 대기: 렌더링 지연 커버 (유지)
- 비즈니스 복구: 화면 상태 오류 커버 (신규)
- 두 레이어가 결합하여 확실한 복구
