# AIMS 티어 및 크레딧 정책

## 개요

AIMS는 **스토리지 + 크레딧** 2가지 리소스로 단순화된 과금 체계를 사용합니다.

- **스토리지**: 문서 저장 공간 (GB)
- **크레딧**: OCR + AI 통합 사용량

---

## 크레딧 시스템

### 왜 크레딧인가?

| 문제 | 해결 |
|------|------|
| OCR 페이지 노출 → Upstage 가격 비교 | 크레딧으로 추상화 |
| AI 토큰 노출 → OpenAI 가격 비교 | 크레딧으로 추상화 |
| OCR/AI 별도 제한 → 복잡함 | 통합 크레딧으로 단순화 |
| 사용 패턴별 불공정 | 실제 사용량 기반 차감 |

### 크레딧 환산 기준

| 리소스 | 원가 | 크레딧 소모 |
|--------|------|------------|
| OCR 1페이지 | ~2.2원 ($0.0015) | **2 크레딧** |
| AI 1K 토큰 | ~0.5원 ($0.0003) | **0.5 크레딧** |
| 문서 요약/임베딩 | ~0.5원 | **1 크레딧/문서** |

### 환산 공식

```
1 크레딧 = 약 1원 (내부 원가 기준)
1 크레딧 = 2,000 AI 토큰
1 크레딧 = 0.5 OCR 페이지
```

### 크레딧 사용 예시

**일반 티어 (3,000 크레딧) 월간 사용:**

| 사용 패턴 | OCR | AI | 크레딧 |
|----------|-----|-----|--------|
| 문서 중심 | 1,200p (2,400) | 1.2M (600) | 3,000 |
| AI 중심 | 300p (600) | 4.8M (2,400) | 3,000 |
| 균형 | 750p (1,500) | 3M (1,500) | 3,000 |

→ **사용자가 OCR/AI 비율을 자유롭게 조절 가능**

| 패키지 | 크레딧 | 가격 | 크레딧당 |
|--------|--------|------|----------|

**→ 볼륨 디스카운트: 많이 살수록 저렴**

## 크레딧 정책

### 이월 정책

| 크레딧 유형 | 이월 | 유효기간 |
|------------|------|----------|
| 월정액 포함 크레딧 | ❌ 불가 | 해당 월 |
| 추가 충전 크레딧 | ✅ 가능 | 무제한 |

### 사용 순서

1. 월정액 크레딧 먼저 소모
2. 월정액 소진 후 충전 크레딧 사용

### 초과 시 처리

- 크레딧 소진 시 OCR/AI 기능 제한
- 문서 조회, 다운로드는 가능
- 추가 충전 또는 티어 업그레이드 안내
