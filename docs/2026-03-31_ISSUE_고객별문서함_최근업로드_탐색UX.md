# ISSUE: 고객별 문서함에서 최근 업로드 파일 탐색 어려움

- **발견일**: 2026-03-31
- **심각도**: Medium (UX)
- **상태**: Open
- **영향 범위**: 고객별 문서함 사용성

---

## 현상

고객별 문서함에서 최근 업로드한 파일들이 어느 분류 경로에 배치되었는지 쉽게 확인할 수 없음.

### 구체적 상황
- 4개 파일을 업로드함
- 각 파일이 서로 다른 문서 유형으로 분류되어 트리의 여러 곳에 흩어짐
- 날짜 정렬을 해도 **펼쳐진 트리 노드 내에서만** 정렬되어 전체적으로 최근 파일을 찾기 어려움

### 전체 문서 보기와의 차이
- **전체 문서 보기**: 플랫 리스트 → 날짜 정렬로 최근 업로드 파일을 즉시 확인 가능
- **고객별 문서함**: 트리 구조 → 분류별로 흩어져 있어 최근 파일 위치 파악 불가

## 해결 방향

### 방안 1: "최근 업로드" 하이라이트
- 최근 N시간/오늘 업로드된 문서에 시각적 마커(뱃지, 배경색) 추가
- 트리를 탐색하면서 어떤 노드에 최근 문서가 있는지 한눈에 파악 가능
- 고객 노드/분류 노드에 "N건 신규" 배지 표시

### 방안 2: "오늘 업로드" 빠른 필터 강화
- 기존 "오늘" 빠른 필터가 있으나, 트리 구조에서 효과적으로 동작하는지 검증
- 필터 적용 시 최근 문서가 포함된 노드만 자동 펼침

### 방안 3: 업로드 직후 임시 플랫 뷰
- 파일 업로드 직후 "방금 업로드한 파일" 섹션을 트리 상단에 임시 표시
- 각 파일의 분류 경로를 함께 표시 (예: "보험금청구 > 보험금청구서")
- 일정 시간 후 또는 사용자가 닫으면 자동 제거

### 방안 4: 날짜별 그룹 모드에서 정렬 개선
- 현재 문서유형별 그룹이 기본인데, 날짜별 그룹 모드에서 최근 문서 우선 표시
- 또는 "최근 업로드순" 정렬 시 모든 트리 노드를 펼치고 전체 문서를 날짜순으로 재배열
