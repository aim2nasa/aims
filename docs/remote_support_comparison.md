## 1. 검토 배경 및 요구사항

### AIMS 특수 조건
- **자체 호스팅 필수**: 보험 고객 개인정보(계약자 정보, 주민번호, 보험료 납입 내역 등)가 외부 서버를 경유할 수 없음
- **개인정보보호법 준수**: 국외 이전 미동의 시 해외 공용 서버 사용 불가
- **설계사 친화적**: 비전문가인 설계사가 쉽게 사용 가능해야 함
- **보안 양호**: 알려진 미패치 취약점 없을 것

### 인프라 환경
- 서버: TARS (tars.giize.com, Ubuntu, 7.8GB RAM)
- MongoDB, nginx, Redmine, n8n, Qdrant, FastAPI 운영 중
- Tailscale VPN으로 내부 보안 통신

## 2. 후보 솔루션 개요

### 2-1. RustDesk
- Rust로 구현된 오픈소스 원격 데스크톱 (TeamViewer 대체)
- 자체 릴레이 서버(hbbs/hbbr) 운영 가능
- 라이선스: AGPL-3.0 (무료) / 상용 Pro

### 2-2. MeshCentral
- Node.js 기반 웹 원격 관리 플랫폼
- 에이전트 기반 구조 (아웃바운드 443 연결)
- 라이선스: Apache 2.0 (완전 무료)

## 5. 검토한 다른 솔루션들

### 5-1. RustDesk 공식 클라우드 서버 (무료)
- **제외 이유**: 보험 고객 화면이 해외 공용 서버 경유 → 개인정보보호법 위반 위험, 금융감독원 보안 가이드라인 저촉 가능

### 5-2. Apache Guacamole
- 관리자 측 클라이언트 설치 불필요 (브라우저만으로 접속)
- 설계사 PC에 VNC 또는 RDP 서버 활성화 필요
- Windows 10 Home 에디션은 RDP 호스트 기능 없음
- **제외 이유**: 설계사 PC 설정 변경이 필요해 비전문가 대응 어려움

### 5-3. DWService
- 브라우저 기반, 에이전트 설치 없이 1회성 지원 가능
- **제외 이유**: 자체 호스팅 불가 (클라우드 전용) → 보험 데이터 외부 서버 경유 문제

### 5-4. Getscreen.me
- 링크 클릭만으로 즉시 연결, 자체 호스팅 옵션 있음
- **제외 이유**: 자체 호스팅은 유료 (가격 비공개), 오픈소스 아님

### 5-5. NoMachine
- NX 프로토콜로 화면 품질 업계 최고 수준
- **제외 이유**: 설계사 측 클라이언트 설치 필요, 기업 용도 유료

## 7. MeshCentral 보안 강화(하드닝) 체크리스트

> 하드닝: 기본 설치 상태의 보안 허점을 하나씩 더 안전하게 바꾸는 작업

### 서버 측 (TARS)

```json
// /opt/meshcentral/meshcentral-data/config.json 핵심 설정
{
  "settings": {
    "cert": "tars.giize.com",
    "port": 443,
    "redirPort": 80,
    "TLSOffload": false,
    "AllowFraming": false,
    "SelfUpdate": false,
    "AutoBackup": {
      "backupPath": "/opt/meshcentral/backups",
      "backupIntervalHours": 24,
      "keepLastDaysBackup": 10
    }
  },
  "domains": {
    "": {
      "passwordRequirements": {
        "min": 8,
        "upper": 1,
        "lower": 1,
        "numeric": 1,
        "nonalpha": 1,
        "force2factor": true
      }
    }
  }
}
```

| 항목 | 설정 | 이유 |
|------|------|------|
| HTTPS | Let's Encrypt 자동 갱신 | 평문 전송 방지 |
| 2FA 강제 | `force2factor: true` | 비밀번호 탈취만으로 접근 불가 |
| 비밀번호 정책 | 8자 이상 + 대소문자 + 숫자 + 특수문자 | 무차별 대입 공격 방어 |
| 포트 제한 | UFW: 443만 외부 허용 | 불필요한 공격 표면 제거 |
| 자동 백업 | 24시간마다, 10일 보관 | 데이터 손실 방지 |
| 자동 업데이트 비활성 | `SelfUpdate: false` | 의도치 않은 버전 변경 방지 |

### 에이전트 측 (설계사 PC)
- 에이전트는 아웃바운드 443만 사용 → 설계사 방화벽 설정 불필요
- 에이전트 설치 링크는 **설계사별 개인 링크** 생성 (공용 링크 사용 금지)
- MeshCentral 서버에서 설계사별 접근 권한 개별 관리

### 주의사항
- `force2factor` 설정 시 로그인 직후가 아닌 첫 리소스 접근 시 2FA 설정 강제됨 (설계 상 제한)
- MeshCentral 서버 자체가 침해되면 모든 에이전트 접근 가능 → **서버 보안이 가장 중요**
- npm 정기 업데이트 필요: `npm update -g meshcentral`
