# E2E 기능 테스트 시나리오 스킬

Playwright MCP를 사용하여 AIMS 개발 서버의 기능을 테스트합니다.
모든 테스트는 `docs/E2E_TEST_SCENARIOS.md`를 기준으로 수행합니다.

## 트리거

- `/e2e-scenario` (사용자 호출)
- "E2E 테스트", "기능 테스트", "시나리오 테스트", "브라우저 테스트"

## 사전 조건

- **프론트엔드 dev 서버** 실행 중 (port 5177)
- **Playwright MCP** `--ignore-https-errors` 옵션 적용
- **절대 프로덕션(aims.giize.com) 접속 금지** — 사용자 명시적 지시 없이는 localhost만

## 실행 모드

### 1. 기본 실행: `/e2e-scenario`

전체 시나리오를 레벨별로 실행합니다.

```
1. docs/E2E_TEST_SCENARIOS.md 읽기
2. 사용자에게 실행 범위 확인:
   - 전체 (모든 레벨, 모든 영역)
   - 특정 레벨 (예: L1만, L2만)
   - 특정 영역 (예: "3. 고객 목록"만)
   - 미구현 항목만
3. 로그인 (이미 로그인 상태면 스킵)
4. 시나리오 순차 실행
5. 결과 보고
```

### 2. 레벨 지정: `/e2e-scenario L2`

특정 레벨만 실행합니다.

### 3. 영역 지정: `/e2e-scenario 고객`

특정 기능 영역만 실행합니다.

### 4. 누락 분석: `/e2e-scenario --gaps`

현재 시나리오의 빈 곳을 분석하고 추가 시나리오를 제안합니다.

```
1. E2E_TEST_SCENARIOS.md 읽기
2. 프론트엔드 코드에서 라우트/뷰 전수 조사
3. 시나리오에 없는 기능 식별
4. 추가 시나리오 제안 (레벨별)
```

## 로그인 절차

로그인이 필요한 경우 아래 순서로 진행합니다.
**절대 사용자에게 로그인을 시키지 않습니다. Claude가 직접 수행합니다.**

```
1. https://localhost:5177 접속
2. "카카오 로그인" 클릭
3. 카카오 계정 페이지에서:
   - 이메일: .playwright-mcp/kakao.txt 1행
   - 비밀번호: .playwright-mcp/kakao.txt 2행
4. "로그인" 클릭
5. PIN 입력 화면에서:
   - PIN: .playwright-mcp/kakao.txt 3행 (키보드 1자씩 입력)
6. 메인 화면 진입 확인 (URL에 login 없음)
```

## 테스트 실행 절차

각 시나리오 항목별로:

```
1. snapshot 또는 navigate로 현재 상태 확인
2. 해당 액션 수행 (클릭, 입력, 등)
3. 결과 확인:
   - L1: URL 변경 + 핵심 요소 존재 확인 (snapshot)
   - L2: 액션 후 화면 변화 확인 (snapshot + screenshot)
   - L3: 화면 데이터 vs DB 조회 비교 (MongoDB MCP 활용)
   - L4: 에러 메시지/빈 상태 처리 확인
4. PASS/FAIL 판정
5. FAIL 시 스크린샷 저장 + 원인 기록
```

## 결과 보고

테스트 완료 후 아래 형태로 보고합니다:

```markdown
## E2E 테스트 결과 — [영역/레벨]

| # | 시나리오 | 레벨 | 결과 | 비고 |
|---|---------|------|------|------|
| 1.1 | 카카오 로그인 | L1 | PASS | — |
| 3.2 | 전체 고객 — 초성 필터 | L2 | FAIL | ㄲ 클릭 시 빈 결과 미표시 |
| ... | ... | ... | ... | ... |

**전체: X개 항목 | PASS: Y | FAIL: Z | SKIP: W**
```

FAIL 항목이 있으면:
- 스크린샷 경로 표시
- 재현 절차 기록
- 버그 심각도 판단 (Critical/Major/Minor)

## 시나리오 업데이트

테스트 중 새로운 시나리오가 필요하면:

```
1. 사용자에게 "X 시나리오를 추가할까요?" 확인
2. 승인 시 E2E_TEST_SCENARIOS.md에 추가
3. 변경 이력 테이블에 버전+날짜+내용 기록
4. 패치 버전 자동 증가 (v1.0.0 → v1.0.1)
```

버전 규칙:
- **패치** (v1.0.x): 시나리오 항목 추가/수정
- **마이너** (v1.x.0): 새 기능 영역 추가 또는 레벨 전체 구현
- **메이저** (vx.0.0): 문서 구조 변경

## 주의사항

- **Playwright 세션 유지**: 테스트 중 브라우저를 닫거나 로그아웃하지 않음
- **데이터 변경 최소화**: 읽기 테스트 위주. 쓰기(등록/수정/삭제)는 사용자 확인 후 실행
- **프로덕션 접속 절대 금지**: localhost:5177만 사용
- **스크린샷 보존**: FAIL 항목은 반드시 스크린샷 저장
- **MongoDB MCP**: L3 데이터 정합성 검증 시 활용 (읽기 전용)
