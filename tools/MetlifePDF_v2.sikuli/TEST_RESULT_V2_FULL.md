## v2 개선사항 적용 이력

### 1. 미리보기 첫 시도 실패 해결 (7feb9bbb)
- **문제**: 고객통합뷰 최초 진입 시 JS 이벤트 핸들러 미바인딩으로 미리보기 버튼 클릭 무반응
- **해결**: 첫 시도에 12초 안정화 대기 추가
- **결과**: 버튼 클릭 자체의 실패 제거

### 2. PDF 저장 경로 Home키 삽입 (c6591f97)
- **문제**: Ctrl+A로 기존 파일명 전체 선택 후 paste → MetLife 기본 파일명 손실
- **해결**: Home키로 커서를 맨 앞으로 이동 후 경로 prefix만 삽입
- **결과**: MetLife 원본 파일명 보존 + 지정 경로 저장

### 3. 미리보기 타임아웃 최적화 (ea91b9aa)
- **문제**: 첫 시도 30초 타임아웃 중 PDF 렌더링에 35~45초 소요되는 케이스 존재
- **해결**: 첫 시도 타임아웃 30초 → 50초 증가 (`[50, 60, 120]`)
- **상태**: 커밋 완료, 대규모 테스트 미검증 (기능 영향 없음, 성능 최적화)

## OCR 관련 참고

현재 스크롤 페이지별 고객 인식에 Upstage Enhanced OCR API를 사용 중.

### OCR 대안 실험 (2026-02-09)

MetLife 사이트에서 OCR 없이 고객명을 읽을 수 있는지 SikuliX 프로브 스크립트로 실험.

**실험 방법 및 결과:**

| 방법 | 동작 | 결과 |
|------|------|------|
| 더블클릭 + Ctrl+C | 고객명 셀 더블클릭 → Ctrl+C | **실패** - 클립보드 변경 없음 |
| 클릭 + Ctrl+A + Ctrl+C | 셀 클릭 → 전체 선택 → 복사 | **실패** - 클립보드 변경 없음 |
| SikuliX Region.text() | Tesseract 내장 OCR | **에러** - 한국어 traineddata 미설치 |

**결론: MetLife 사이트는 테이블 셀의 텍스트 선택/복사를 차단함.**
- CSS `user-select: none` 또는 커스텀 이벤트 핸들러로 차단 추정
- 클립보드 방식은 사용 불가
- **현재 Upstage OCR 방식 유지가 최선** (페이지당 0.6~3.4초, 높은 정확도)

### No-OCR 모드 실험 (2026-02-09)

OCR 없이 순차 클릭만으로 고객 처리가 가능한지 실험. `--no-ocr` 옵션 구현 후 ㅋ 초성 그룹 대상 검증.

**구현 방식:**
- `--no-ocr` CLI 옵션 추가
- OCR 대신 ROW_01~ROW_N 가상 고객명 생성 (blind sequential clicking)
- 빈 행 감지: 행 클릭 후 고객 목록 헤더(`IMG_CUSTNAME`)가 보이면 → 아직 목록 페이지 = 빈 행
- 스크롤 중복 감지 비활성화 (고객명 비교 불가)

**실험 결과:**

| 항목 | No-OCR 모드 |
|------|-------------|
| 명령 | `--chosung ㅋ --integrated-view --no-ocr` |
| 처리 행 수 | 8행 (모두 개별 처리) |
| 에러 | 0 |
| PDF 저장 | 2건 |
| 소요 시간 | 14분 45초 |
| 빈 행 감지 | ROW_09에서 정상 감지 |

**행별 상세:**

| 행 | 실제 고객 | 변액리포트 | 결과 |
|----|----------|-----------|------|
| ROW_01 | 캐치앤코리아 | 있음 | PDF 저장 완료 |
| ROW_02 | 코우머스 | 없음 | 변액 없음 확인 |
| ROW_03 | 코우머스 | 없음 | 변액 없음 확인 |
| ROW_04 | 코우머스 | 없음 | 변액 없음 확인 |
| ROW_05 | 코우머스 | 없음 | 변액 없음 확인 |
| ROW_06 | 코우머스 | 없음 | 변액 없음 확인 |
| ROW_07 | 코데바이스코리아 | 없음 | 변액 없음 확인 |
| ROW_08 | 키움 | 있음 | PDF 저장 완료 |
| ROW_09 | (빈 행) | - | 빈 행 감지 → 중단 |

**참고:** 같은 고객명이 여러 행에 나오더라도 각각 별도의 행이므로 모두 개별 방문하여 처리해야 함.

**결론:**
- No-OCR 모드는 기능적으로 정상 동작 (에러 0, 빈 행 감지 정상)
- OCR 없이도 모든 행을 순차적으로 처리 가능
- No-OCR 모드는 OCR API 장애 시 fallback 용도로 보존
