## v2 개선사항 적용 이력

### 1. 미리보기 첫 시도 실패 해결 (7feb9bbb)
- **문제**: 고객통합뷰 최초 진입 시 JS 이벤트 핸들러 미바인딩으로 미리보기 버튼 클릭 무반응
- **해결**: 첫 시도에 12초 안정화 대기 추가
- **결과**: 버튼 클릭 자체의 실패 제거

### 2. PDF 저장 경로 Home키 삽입 (c6591f97)
- **문제**: Ctrl+A로 기존 파일명 전체 선택 후 paste → MetLife 기본 파일명 손실
- **해결**: Home키로 커서를 맨 앞으로 이동 후 경로 prefix만 삽입
- **결과**: MetLife 원본 파일명 보존 + 지정 경로 저장

### 3. 미리보기 타임아웃 최적화 (ea91b9aa)
- **문제**: 첫 시도 30초 타임아웃 중 PDF 렌더링에 35~45초 소요되는 케이스 존재
- **해결**: 첫 시도 타임아웃 30초 → 50초 증가 (`[50, 60, 120]`)
- **상태**: 커밋 완료, 대규모 테스트 미검증 (기능 영향 없음, 성능 최적화)

## 미리보기 성공률 분석

| 구분 | 상세 |
|------|------|
| 첫 시도 성공 | 대부분 (12초 안정화 후 PDF 로딩 30초 이내 완료) |
| 첫 시도 실패 → 재시도 성공 | 일부 (PDF 렌더링 35~45초 소요 케이스) |
| 최종 실패 | 0건 (3회 재시도 메커니즘으로 100% 성공) |

**참고**: 타임아웃 [30, 60, 120] → [50, 60, 120] 변경은 첫 시도 성공률을 높여 전체 소요 시간 단축 목적. 기능적 안정성에는 영향 없음.

## OCR 관련 참고

현재 스크롤 페이지별 고객 인식에 Upstage Enhanced OCR API를 사용 중.

### OCR 대안 실험 (2026-02-09)

MetLife 사이트에서 OCR 없이 고객명을 읽을 수 있는지 SikuliX 프로브 스크립트로 실험.

**실험 방법 및 결과:**

| 방법 | 동작 | 결과 |
|------|------|------|
| 더블클릭 + Ctrl+C | 고객명 셀 더블클릭 → Ctrl+C | **실패** - 클립보드 변경 없음 |
| 클릭 + Ctrl+A + Ctrl+C | 셀 클릭 → 전체 선택 → 복사 | **실패** - 클립보드 변경 없음 |
| SikuliX Region.text() | Tesseract 내장 OCR | **에러** - 한국어 traineddata 미설치 |

**결론: MetLife 사이트는 테이블 셀의 텍스트 선택/복사를 차단함.**
- CSS `user-select: none` 또는 커스텀 이벤트 핸들러로 차단 추정
- 클립보드 방식은 사용 불가
- **현재 Upstage OCR 방식 유지가 최선** (페이지당 0.6~3.4초, 높은 정확도)
