## 코드 문제점

### 현재 코드 (click_all_rows_with_scroll 함수)

```python
# 스크롤 후에는 같은 Y 좌표에 새 행이 표시되므로 clicked_y_set 클리어
clicked_y_set.clear()  # ❌ 문제!
current_y = first_row_y
```

### 문제점

1. **clicked_y_set.clear()**: 스크롤 후 중복 클릭 방지 정보를 모두 삭제
2. **first_row_y 재사용**: 헤더 기준으로 재계산하지만, 그 위치에 이미 클릭한 행이 있음
3. **체크 상태 미확인**: 이미 체크된 행인지 확인하지 않고 무조건 클릭

## 해결 방안

### 방안 1: 체크박스 상태 확인 (권장)

```python
# 스크롤 후: 이미 체크된 행은 스킵
while current_y <= visible_bottom_y:
    if is_row_checked(base_x, current_y):
        log("이미 체크됨 - 스킵")
        current_y += ROW_HEIGHT
        continue

    # 체크 안 된 행만 클릭
    click(Location(base_x, current_y))
    ...
```

### 방안 2: 스크롤 양 조절

```python
# 스크롤 시 마지막 클릭 행이 화면에서 사라지도록 충분히 스크롤
# 현재: wheel(WHEEL_DOWN, 20)
# 개선: 정확히 VISIBLE_ROWS 만큼 스크롤
wheel(WHEEL_DOWN, VISIBLE_ROWS * 5)  # 7행 * 5 = 35 notches
```

### 방안 3: 증권번호 기반 추적

```python
# 클릭한 증권번호를 Set으로 추적
clicked_policy_numbers = set()

# 클릭 전 증권번호 확인
policy_number = extract_policy_number(current_row)
if policy_number in clicked_policy_numbers:
    continue  # 스킵
```

`click_all_rows_with_scroll()` 함수 내 스크롤 후 처리 부분

### 수정 내용

```python
# clicked_y_set.clear()  # ❌ 삭제
current_y = first_row_y

# 이미 체크된 행은 스킵하는 로직 추가
while current_y <= visible_bottom_y:
    # ★ 핵심 수정: 체크 상태 확인
    if is_row_checked(base_x, current_y):
        log(u"    Y=%d 이미 체크됨 - 스킵" % current_y)
        current_y += ROW_HEIGHT
        continue

    # 이하 기존 로직...
```
