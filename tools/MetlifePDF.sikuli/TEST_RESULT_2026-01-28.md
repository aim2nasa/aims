## 기술적 구현 상세

### 종료 조건
- `MAX_SCROLLS = 1`: 스크롤 1회 후 종료
- 13개 보고서, 화면 7개 표시 → 첫 화면 7개 + 스크롤 후 6~7개 = 13~14개

### Y좌표 추적
- 클릭한 Y좌표를 `clicked_y_set`에 저장
- 스크롤 후 `clicked_y_set.clear()`로 초기화 (새 행이 같은 Y에 나타남)

### 스크롤 후 처리
1. 체크된 행 해제 (중복 선택 방지)
2. Y좌표 추적 클리어
3. unchecked 체크박스 재탐색

### 단계별 상세

#### [1] 테스트 실행

```bash
# SikuliX IDE에서 실행
java -jar C:\SikuliX\sikulixide-2.0.5.jar -r verify_customer_integrated_view.py
```

스크립트가 수행하는 작업:
1. 고객통합뷰 진입
2. 변액보험리포트 모달 열기
3. 각 행 순회하며 클릭
4. 클릭할 때마다 스크린샷 저장
5. 스크롤 전/후 스크린샷 저장
6. 로그 파일 생성

#### [2] 로그 수집

로그 파일: `debug_log.txt`

주요 분석 포인트:
- `[탐색] unchecked 체크박스 N개 발견` - 현재 화면의 클릭 가능한 행
- `[N] 행 클릭 성공 (X, Y)` - 실제 클릭된 위치
- `[스크롤 N/M]` - 스크롤 진행 상황
- `[END]` - 종료 조건

#### [3] 스크린샷 분석

스크린샷 위치: `screenshots/`

| 파일 패턴 | 용도 |
|-----------|------|
| `modal_XX.png` | 모달 전체 + 선택된 행 마커 (증권번호 확인용) |
| `row_XX.png` | 선택된 행 영역만 (행 내용 상세 확인) |
| `scroll_before_XX.png` | 스크롤 전 화면 상태 |
| `scroll_after_XX.png` | 스크롤 후 화면 상태 |

**증권번호 추출 방법:**
1. `modal_XX.png` 이미지 열기
2. 선택된 행(빨간 마커)에서 증권번호 읽기
3. 10자리 숫자 형태: `00XXXXXXXX`

분석 결과 테이블 생성:


집계 항목:

- **중복 클릭**: 총 클릭 - 고유 보고서
- **누락된 보고서**: 전체 보고서 - 고유 보고서

#### [5] 결과 검증

**핵심 원칙 검증:**

| 검증 항목 | 기준 | 판정 |
|-----------|------|------|
| 모든 보고서 클릭 | 고유 보고서 수 = 전체 보고서 수 | PASS/FAIL |
| 중복 최소화 | 중복 클릭 ≤ 2회 | PASS/FAIL |
| 정상 종료 | 스크롤 제한 도달 후 종료 | PASS/FAIL |

**실패 시 대응:**

| 실패 유형 | 원인 분석 | 수정 방향 |
|-----------|-----------|-----------|
| 보고서 누락 | Y좌표 추적 오류 | `clicked_y_set.clear()` 타이밍 검토 |
| 과다 중복 | 종료 조건 미흡 | `MAX_SCROLLS` 조정 |
| 무한 루프 | 종료 조건 없음 | 스크롤 횟수 제한 추가 |

### 자동화 가능성

현재는 수동으로 스크린샷 분석 및 증권번호 추출을 수행하지만, 다음 방법으로 자동화 가능:

1. **OCR 기반**: Tesseract로 modal 스크린샷에서 증권번호 자동 추출
2. **픽셀 해시**: 증권번호 영역만 crop하여 해시 비교 (전체 모달보다 정확)
3. **API 연동**: 백엔드 API에서 클릭된 보고서 ID 로깅
