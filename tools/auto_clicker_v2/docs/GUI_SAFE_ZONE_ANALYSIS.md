# AutoClicker v2 GUI - SikuliX 안전 영역 분석

> 분석일: 2026-02-10
> 대상: AutoClicker v2 컴팩트 모드 GUI가 SikuliX 동작에 미치는 영향

## 결론

**AutoClicker GUI를 MetLife와 같은 모니터에 표시해도 SikuliX 동작에 문제 없음.**

---

## 1. 컴팩트 모드 GUI 위치

실행 시 `_start()` → 자동 `_enter_compact()` 전환.

```
크기: 850 x 47 px
위치: 화면 우측 하단 (작업표시줄 바로 위)
좌표: X = work_right - 850, Y = taskbar_y - 47
     → X: 1070~1920, Y: 985~1032 (1920x1080 기준)
속성: overrideredirect(True), topmost(True)
```

## 2. SikuliX 작업 영역

### OCR 캡처 영역

```
TABLE_REGION = Region(20, 362, 1890, 590)
→ X: 20~1910, Y: 362~952
```

전체 화면 캡처 후 이 영역으로 크롭하여 OCR API에 전송.

## 4. 충돌 검증

| 검증 항목 | SikuliX 영역 | GUI 영역 | 간격 | 판정 |
|-----------|-------------|----------|------|------|
| OCR 캡처 | Y: 362~952 | Y: 985~1032 | 33px | **안전** |
| 행 클릭 (최대 13행) | Y: 471~915 | Y: 985~1032 | 70px | **안전** |
| Page Down 포커스 | (310, 809) | X: 1070~1920 | X축 불일치 | **안전** |
| 고객통합뷰 버튼 | (~1781, 246) | Y: 985~1032 | 739px | **안전** |
| "다음" 버튼 | Y: 955~985 | Y: 985~1032 | 0px (접함) | **안전** (아래 상세) |
| 이미지 매칭 간섭 | 전체 화면 스캔 | 화면에 표시 | - | **안전** (아래 상세) |

### "다음" 버튼 상세

- "다음" 버튼(Y: 955~985)과 GUI(Y: 985~)가 정확히 맞닿음
- GUI가 버튼 위를 **가리지 않음** (GUI는 Y=985 이후 시작)
- X축도 다름: "다음" 버튼은 테이블 중앙~좌측, GUI는 X=1070 이후
- "다음" 버튼은 이미지 매칭(`exists()`)으로 탐색하므로 좌표 오차 무관

### 이미지 매칭 간섭 상세

- SikuliX `find()`/`exists()`는 전체 화면에서 이미지 패턴 검색
- AutoClicker GUI: 어두운 회색(#2b2b2b) 배경 + 작은 한글 텍스트
- MetLife 버튼들: 밝은 파란색/흰색 UI 요소
- 시각적으로 완전히 다른 패턴 → **오인식 가능성 없음**

## 5. 안전 마진 요약

```
OCR 캡처 하단(Y=952) ─── 33px ──→ GUI 상단(Y=985)
행 클릭 최하단(Y=915) ── 70px ──→ GUI 상단(Y=985)

```
